## 💡 BOJ 2023 - 신기한 소수 [[문제 링크](https://www.acmicpc.net/problem/2023)]

| 항목 | 내용 |
|------|------|
| 문제 유형 | `Recursion` |
| 사용 알고리즘 | `가지치기` |
| 난이도 | `골드5` |

### ✅ 문제 설명
**[문제 요약]**

- 신기한 소수란, 왼쪽부터 1자리, 2자리, 3자리, 4자리 .. 수 모두 소수인 수
- N자리의 숫자 중에서 신기한 소수를 모두 출력하라

**[제약 사항]**

- N <= 8

### 🔍 풀이 접근

- 소수 테이블을 갖고 있다고 하자
- Brute-Force하게 숫자를 만들고 소수 판별을 진행하면 된다

**[설계]**

- 에라토스테네스의 채로 소수 테이블을 만든다
- N자리 숫자를 모두 만들어보는데, 더 이상 소수가 아니라면 해당 path는 넘어간다 (가지치기)

**[설계 미스]**

- 메모리 제한이 4MB라서 1억개의 bool 테이블을 만들 수 없음 (100MB)
- 그냥 vector에 저장해놓고, vector에서 해당 숫자를 찾는 식으로 수정하자
- vector에서 해당하는 숫자가 있는지 찾는 건 이분탐색 혹은 BBST를 활용하면 된다
- 여기서는 이분탐색을 활용해보자

### 💡 핵심 로직
- 8자리까지 소수 구하는데 오래걸림
- 어차피 신기한 소수로 시작할 수 있는건 2, 3, 5, 7밖에 없음
- 가지치기로 충분한가? 들어갈 때 소수 검사해버리고

### 📌 배운 점
- 소수 판별법과 가지치기로 시간복잡도가 줄어드는 문제를 해결하는 경험을 했습니다.
