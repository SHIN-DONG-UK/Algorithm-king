## 💡 BOJ 12101 - 1,2,3 더하기2 [[문제 링크]](https://www.acmicpc.net/problem/12101)

| **항목** | **내용** |
| - | - |
| **문제 유형** | `Recursion` |
| **사용 알고리즘** | |
| **난이도** | `실버1` |

### ✅ **문제 설명**`
**[문제 요약]**

- 정수 n과 k가 주어졌을 때, n을 1, 2, 3의 합으로 나타내는 방법 중에서 k번째로 오는 식을 구하라
- 4를 1, 2, 3의 합으로 나타내보자

1+1+1+1
1+1+2
1+2+1
2+1+1
2+2
1+3
3+1

- 사전 순으로 정렬하면
1+1+1+1
1+1+2
1+2+1
1+3
2+1+1
2+2
3+1

**[제약 사항]+**

- n <= 11
- k <= 2^31 -1
- k번째 식이 없는 경우, -1 출력

### 🔍 **풀이 접근**

- n이 11이하이므로, Brute-Force하게 접근해도 상관없다
- 3^11 = 177,147
- 모든 경우를 다 만들어보고, k번째가 되면 해당 순서열을 출력하는 식으로 문제를 풀어도 되겠다
- 만약 모든 경우를 다 만들었는데도 k번째가 안되면, -1을 출력하면 되겠다
- 재귀적으로 함수를 구성하면 구현이 쉽겠다

### 💡 **핵심 로직**

- 매번 1부터 3까지 숫자를 사용해본다
- 만약 현재 숫자를 사용했을 때 n보다 작으면, 다음 depth로 넘어간다
- n보다 크다면, 현재 depth에서 나온다
- n과 같다면, 카운트한다
   - 만약 카운트가 k와 같다면, 현재 저장된 path를 출력하고 재귀를 탈출한다
