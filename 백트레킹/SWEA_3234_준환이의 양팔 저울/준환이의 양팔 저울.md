# 1. 이해

```
[상황]
- N개의 서로 다른 무게를 가진 무게 추
- 양팔저울
- 모든 무게 추를 양팔저울에 올리는 순서는 총 N!
- 여기에 왼쪽에 올릴지 오른쪽에 올릴지 -> 2^N * N!
- 단, 오른쪽 저울 무게가 왼쪽보다 크면 안됨

[출력]
- 이런 방법으로 준환이가 양팔 저울에 모든 무게추를 올리는 방법은 몇 가지?

[입력]
- Test Case T
- N
- 서로 다른 무게의 무게 추
```

# 2. 설계

```
[알고리즘]
- 이 문제는 어떻게 하면 되는지 알려주고 있음
- 그냥 백트레킹 하면 됨
- 무게가 더 커지면 그 노드로 안가게 하면 됨

[필요한 것들]
- dfs를 구성할 때 왼쪽 파라미터, 오른쪽 파라미터가 필요할 듯
- 기본적으로 순열 구조를 따라야 할 것
```

# 3. 문제의 핵심

```
- 문제대로 2^N * N!을 구현하면 시간초과가 발생한다
- 사실 모든 노드를 탐색할 필요가 없다
- 우리는 수식으로 2^N * N!을 얻을 수 있다
- 그래서 수식으로 구해도 되는 구간이 나오면, 수식으로 구하면 된다
- 그렇지 않은 구간을 그래프 탐색하도록 코드를 작성하여야 한다
- 수식으로 구해도 되는 구간은 나머지 무게를 모두 right에 더해도
left가 더 크거나 같은 경우이다
```
