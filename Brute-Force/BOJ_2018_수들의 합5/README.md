## 💡 BOJ 2018 - 수들의 합5 [[문제 링크](https://www.acmicpc.net/problem/2018)]

| 항목 | 내용 |
|------|------|
| 문제 유형 | `Brute-Force` |
| 사용 알고리즘 | `투 포인터` |
| 난이도 | `실버5` |

### ✅ 문제 설명

**[문제 요약]**

- 어떠한 자연수 N은 몇 개의 연속된 자연수의 합으로 나타낼 수 있다
- 어떤 자연수 N에 대하여 이 N을 몇 개의 연속된 자연수의 합으로 나타낼 수 있는지 가지수를 알고 싶다
- N을 입력받아 가지수를 출력하라

**[제약 사항]**

- 1 <= N <= 10,000,000

### 🔍 풀이 접근

- 연속된 수의 합을 빠르게 표현하는 방법이 필요
- S(n) = 1 + 2 + ... + n이라고 하자
- 자연수 a에서 b까지의 합은 S(b) - S(a-1)로 표현할 수 있다
- 이제 O(N^2) 방법으로 문제를 해결할 수 있다
- 하지만 TLE다
- 또, 10,000,000길이 long long 배열을 만들면 메모리 초과다
- 투 포인터를 응용하면 문제를 빠르게 해결할 수 있다


### 💡 핵심 로직

- left = 1, right=1, sum=1에서 시작한다
- sum < N이면 right를 늘린다 -> right++
- 그리고나서 sum += right
- sum > N이면 sum -= left
- 그리고나서 left를 늘린다 -> left++
- left <= right이고 right <= N일 때까지 반복한다
