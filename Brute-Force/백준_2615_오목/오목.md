# 1. 이해

```
[오목]
- 검은 바둑알과 흰 바둑알을 교대로 놓기
- 19 x 19
- 5개 연속이면 이김
- 6개 연속이면 아님

[출력]
- 첫 줄에 검은색이 이겼으면 1, 흰색이 이겼으면 2, 아직 승부가 결정되지 않았으면 0
- 둘째 줄에 연속된 다섯 개의 바둑알 중에서 가장 왼쪽 위 위치 출력

[입력]
- 19 x 19 바둑판
```

# 2. 설계

```
1. 바둑판을 돈다
2. 1또는2에서 시작한다
3. 가로세로대각선을 각각 검사한다
	- 오른쪽, 오른쪽 아래, 아래, 왼족 아래만 검사하면 됨
4. 방문처리를 해놔서 중복 연산하는 일이 없도록 한다
```

# 3. 설계 오류

```
1. 왼쪽 위 좌표를 쉽게 구하기 위해 4방향만 검사하려고 했는데, 반례존재
	- 1 1 1 1 1 1 1
	- 얘는 정답이 아닌데, 정답을 출력해버림
	- 방문처리로 해결하려고 하자니 다시 원복하는 코드 필요
2. 그냥 역방향도 탐색하자. 그대신 왼쪽 위 좌표를 구하기 위해
2중 for문을 j -> i로 구성하자
```
