## 💡 BOJ 3649 - 로봇 프로젝트 [[문제 링크](https://www.acmicpc.net/problem/3649)]

| 항목 | 내용 |
|------|------|
| 문제 유형 | `Brute-Force` |
| 사용 알고리즘 | `투 포인터` |
| 난이도 | `골드5` |

### ✅ 문제 설명
**[문제 요약]**

- 구멍을 막을 두 레고 조각이 필요하다
- 구멍의 너비는 x cm이고, 구멍에 넣을 두 조각의 길이의 합은 구멍의 너비와 정확하게 일치해야 한다
- 구멍을 완벽하게 막을 수 있는 두 조각을 구하라

**[제약 사항]**

- 입력은 여러 개의 테스트 케이스로 이루어져 있다
- 구멍의 너비 1 <= x <= 20
- 레고 조각의 수 0 <= n <= 1,000,000
- 레고 조각의 길이 l은 양의 정수이며, 단위는 나노미터이다. 10cm를 넘지 않는다
- 구멍을 완벽하게 막을 수 있는 두 조각이 없다면 'danger'를 출력
- 막을 수 있는 경우에는 'yes l1 l2'를 출력 (l1 <= l2)
- 정답이 여러 개인 경우에는 |l1- l2|가 가장 큰 것을 출력한다

### 🔍 풀이 접근

- n개의 블록 중 두 개의 합이 x와 같은 두 개의 조각을 찾는 문제이다
- O(N^2)의 반복문으로 모든 경우를 탐색하면 최적해가 보장된다
- 최악의 경우 n = 1,000,000 이므로, O(N^2)의 복잡도는 무리다
- n개의 블록을 길이 오름차순으로 정렬하고, 투 포인터로 탐색하면 문제를 해결할 수 있지 않을까?

### 💡 핵심 로직

- n개의 블록이 정렬되었을 때, 최선은 한 블록은 첫 번째이고 다른 한 블록은 마지막 블록인 경우다
- 그런데 이게 x와 같지 않으면, 둘 중 하나를 바꿔야 한다
- x보다 크면 r_end를 내릴 수 밖에 없다 (이는 r_end로 가능한 모든 조합을 해본 것과 같다)
- x보다 작으면 l_end를 올릴 수 밖에 없다 (이는 l_end로 가능한 모든 조합을 해본 것과 같다)
- x와 같으면 탐색을 종료한다

### 📌 배운 점
- Brute-Force로부터 투 포인터로 사고과정을 옮기는 연습을 했습니다.
- 투 포인터 알고리즘의 정당성에 대해 알게 되었습니다.
