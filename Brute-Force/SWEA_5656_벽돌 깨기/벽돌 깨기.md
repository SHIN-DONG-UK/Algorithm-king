# 1. 이해

```
[상황]
- 구슬을 쏘아 벽돌을 깰 것
- 구슬은 N번만 쓸 수 있음
- w x h 격자에 벽돌 정보
	0 : 빈칸
	그외 : 벽돌

[게임 규칙]
- 구슬은 좌, 우로만 움직일 수 있어서 항상 맨 위에 있는 벽돌만 깨트릴 수 있음
- 벽돌은 숫자 1 ~ 9로 표현
	- 구슬이 명중한 벽돌은 상하좌우로 (벽돌에 적힌 숫자 -1) 칸 만큼 같이 제거
- 제거되는 범위 내에 있는 벽돌은 동시에 제거됨
- 빈 공간이 있을 경우 벽돌은 밑으로 떨어지게 된다
- N개의 벽돌을 떨어트려 최대한 많은 벽돌을 제거하려고 함

[출력]
- N W H 그리고 벽돌의 정보가 주어질 때, 최대한 많은 벽돌을 제거하고 남은 벽돌 개수를 구하라

[입력]
- Test Case
- N ( max 4 ) W ( max 12 ) H ( max 15 )
- 맵
```

# 2. 설계

```
[알고리즘]
- 언제 최대가 될지 모르므로, 모든 경우를 다 해보는 Brute Force를 해야 한다
- 전체적인 시뮬레이션은 Dfs 뼈대에서 진행되도록 한다
	- 리턴되는 깊이는 N이 된다
	- 같은 곳에서 계속 쏠 수 있으므로, 중복순열을 뼈대로 해야 함
- 만약 제거할 맨 위의 벽돌이 결정되었다면, 해당 벽돌을 깨뜨려 연쇄 반응을 수행한다
	- 4방향으로 숫자-1만큼 반복하면서 벽돌을 제거하는데, 1보다 큰 숫자가 나오면 queue에 담아놓는다
	- 제거가 끝나면 queue에서 꺼내어 제거를 반복한다
	- queue가 빌 때까지 해당 과정을 반복한다
- 이렇게 해서 가장 적게 남은 벽돌을 업데이트 한다
```
