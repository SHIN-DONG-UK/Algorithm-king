## 💡 BOJ 16943 - 숫자 재배치[[문제 링크](https://www.acmicpc.net/problem/16943)]

| 항목 | 내용 |
|------|------|
| 문제 유형 | `Brute-Force` |
| 사용 알고리즘 | `백트레킹` |
| 난이도 | `실버1` |

### ✅ 문제 설명
**[문제 요약]**

- 두 정수 A와 B가 있을 때, A에 포함된 숫자의 순서를 섞어서 새로운 수 C를 만들려고 한다
   - 즉, C는 A의 순열 중 하나가 되어야 함
- 가능한 C 중에서 B보다 작으면서, 가장 큰 값을 구하라
- C는 0으로 시작하면 안된다

**[제약 사항]**

- 그러한 C가 없는 경우 -1을 출력
- A, B <= 10억

### 🔍 풀이 접근
- 그리디하게 문제를 해결할 수 있어 보인다 => 반례 확인 => Brute-Force로 풀어야겠다

```
[반례]
- 44123 44321
- 이렇게 했을 때 44312을 만들 수 있어야 하는데, 그리디하게 해버리면 무조건 44321을 만들어버림
```

### 💡 핵심 로직
- 그리디하게 풀려고 했을 때 잘못된 로직
```
- 일단 B보다 작으려면 A의 자릿수가 B보다 크면 안된다
   - if A의 자릿수가 B보다 크다 -> return -1

- B보다 자릿수가 작으면 최댓값을 출력하면 된다
   - if A의 자릿수가 B보다 작다 -> return 최댓값

- A와 B의 자릿수가 같으면 그리디하게 선택한다
   - 그리디하게 선택했을 때, B의 첫 번째 자리(왼쪽에서부터)보다 작은 걸 첫 번째 자리에 둘 수 있으면 그걸 두고 이후로는 최댓값을 출력
   - 그렇지 않으면 다음 자리에서 확인
      - 그리디하게 선택했을 때 B의 다음 자리보다 작은 걸 그 자리에 둘 수 있으면 그걸 두고 이후로는 최댓값 출력
   - 그렇지 않으면 다음 자리 확인
   ...
```
