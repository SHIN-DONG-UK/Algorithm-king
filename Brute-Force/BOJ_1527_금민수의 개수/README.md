## 💡 BOJ 1527 - 금민수의 개수 [[문제 링크](https://www.acmicpc.net/problem/1527)]

| 항목 | 내용 |
|------|------|
| 문제 유형 | `Brute-Force` |
| 사용 알고리즘 | |
| 난이도 | `실버1` |

### ✅ 문제 설명
**[문제 요약]**

- 금민수는 4와 7로만 이루어진 수를 말한다
- A와 B가 주어졌을 때, A보다 크거나 같고, B보다 작거나 같은 자연수 중에 금민수인 것의 개수를 출력하라

**[제약 사항]**

- 1 <= A <= 10억
- A <= B <= 10억

### 🔍 풀이 접근
- Brute-Force하는 문제 아닌가?
- 근데 A보다 길고 B보다 짧은 길이들은 무조건 되니까, 이 부분은 그냥 2^길이 로 더해준다
- A와 길이가 같을 때랑 B와 길이가 같을 때만 Brute-Force하게 만들어서 개수 카운트해주면 되겠다

### 💡 핵심 로직
- A와 B의 길이를 구한다
- A+1 ~ B-1 길이의 애들은 그냥 더해준다
- A길이 Brute-Force한다
- B길이 Brute-Force한다
- 최종 출력한다

### 📌 배운 점
- Brute-Force하게 접근하는 것이 더 효율적인 것을 알고 적용해 문제를 풀이했습니다.
