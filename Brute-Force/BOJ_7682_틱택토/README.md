## 💡 BOJ 7682 - 틱택토 [[문제 링크](https://www.acmicpc.net/problem/7682)]

| 항목 | 내용 |
|------|------|
| 문제 유형 | `Brute-Force` |
| 사용 알고리즘 | `순열` |
| 난이도 | `골드5` |

### ✅ 문제 설명
[문제 요약]

- 틱택토 게임은 두 명의 사람이 번갈아가며 말을 놓는 게임
- 게임판은 3x3 격자판이며, 처음에는 비어 있음
- 첫 번째 사람이 X를 놓고, 두 번째 사람이 O를 놓는다
- 어느 때든지 한 사람의 말이 가로, 세로, 대각선 방향으로 3칸을 잇는 데 성공하면 게임은 즉시 종료
- 게임판이 가득 차도 종료
- 게임판의 상태가 주어지면, 그 상태가 틱택토 게임에서 발생할 수 있는 최종 상태인지 판별하라

[제약 사항]

- 입력은 여러 개의 테스트 케이스로 이루어짐
- 각 줄은 9개의 문자를 포함하며, 'X', 'O', '.' 중 하나
- '.'은 빈칸
- 각 줄이 하나의 테스트 케이스
- 입력의 마지막에는 문자열 'end'가 주어짐

### 🔍 풀이 접근
- 최종 형태로 가능한 모든 경우를 탐색한 다음, 저장해놓는다
- 그리고 이 저장해 놓은 후보들 중에 입력이 존재하는지 ofline query를 수행한다
- 이렇게 했을 때 좋은 점은 두는 순서에 따른 결과를 반영할 수 있음
- 두고나서 검사를 때린 다음 완성됐으면 바로 종료하는 방식으로 

### 💡 핵심 로직
```cpp
void go(int idx) {
	// 1. 조기 종료
	if (check()) {
		hSet.insert(tictacto);
		return;
	}
	// 2. 조기 종료에 해당하지 않으면, 최종 상태
	if (idx == 9) {
		hSet.insert(tictacto);
		return;
	}

	// 3. 순열
	for (int i = 0; i < 9; i++) {
		if (visited[i]) continue;
		visited[i] = true;
		//path[idx] = i; // for debug
		
		if (idx % 2 == 0) {
			tictacto[i] = 'X';
		}
		else {
			tictacto[i] = 'O';
		}
		
		go(idx + 1);
		visited[i] = false;
		tictacto[i] = '.';
	}
}
```

### 📌 배운 점
- 모든 경우를 구한 다음, 이 경우 안에 존재하는지 검사하는 방식을 활용해 많은 분기를 피했습니다.
