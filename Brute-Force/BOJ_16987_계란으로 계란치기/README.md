## 💡 BOJ 16987 - 계란으로 계란치기[[문제 링크](https://www.acmicpc.net/problem/16987)]

| 항목 | 내용 |
|------|------|
| 문제 유형 | `Brute-Force` |
| 사용 알고리즘 | `백트레킹` |
| 난이도 | `골드5` |

### ✅ 문제 설명
**[문제 요약]**

- 각 계란에는 내구도와 무게가 정해져있다
- 계란으로 계란을 치게 되면 각 계란의 내구도는 상대 계란의 무게만큼 깎인다
- 그리고 내구도가 0이하가 되는 순간 계란은 깨진다
- 계란이 일렬로 놓여있을 때, 왼쪽부터 차례로 들어서 한 번씩만 다른 계란을 쳐서 최대한 많은 계란을 깨고싶다
- 일렬로 놓인 계란들의 내구도와 무게가 차례대로 주어질 때, 최대 몇 개의 계란을 깰 수 있는지 출력하라

**[제약 사항]**

- 계란의 수 N: 1 <= N <= 8
- 각 계란의 내구도 1 <= S <= 300
- 각 계란의 무게 1 <= W <= 300

### 🔍 풀이 접근

- 문제의 크기가 매우 작아서, 그냥 Brute-Force하게 접근해도 된다
- O(N!)이어도 문제를 해결할 수 있다

### 💡 핵심 로직

- 현재 계란을 idx라고 하자
- idx 계란을 제외한 모든 계란을 트라이해본다
- 단, 이미 깨진 계란은 그냥 넘어간다
- 트라이하면 현재 계란과 트라이한 계란의 내구도가 변하는데, 트라이하고 나와서는 '원복'해준다
- 이러한 재귀 함수를 작성해보자

### 📌 배운 점
- 단순한 예외 처리가 반례를 만들어 낸다는 것을 배웠습니다.
