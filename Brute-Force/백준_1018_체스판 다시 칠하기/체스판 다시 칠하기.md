# 1. 이해

```
[맵]
- N x M (8 ≤ M,N ≤ 50)

[행동]
- 8x8 크기로 잘라낸 후 체스판을 만들기 위해 색칠

[출력]
- 칠해야 하는 최소 개수
```

# 2. 설계

```
1. 언제 최소가 될지 알 수 없음
2. 걍 다해보면 됨
	- 왼쪽 모서리가 될 수 있는 범위를 찾고, 이 범위만큼 for문을 돈다
	- 모서리 범위 : 
		a.행 : 0 ~ N-8
		b.열 : 0 ~ M-8
	- 체스판이 될 수 있는 규칙은 뭘까?
		a.왼쪽 모서리가 흰색인 경우
			행+열이 짝수면 흰색, 홀수면 검은색이어야 함
		b.왼쪽 모서리가 검은색인 경우
			행+열이 짝수면 검은색, 홀수면 흰색이어야 함
3. 맵을 돌면서 8x8로 자르고, 칠하는 개수 구하고 최솟값 업데이트

```

# 3. 설계 오류

```
- 왼쪽 모서리는 항상 짝수가 아님
- 그래서 왼쪽 모서리가 짝수인지 홀수인지 체크 필요!!
- 그래서 왼쪽 모서리가 짝수이면 행+열이 짝수인 애들이랑 색깔이 같아야 하고,
홀수이면 행+열이 홀수인 애들이랑 색깔이 같아야 함
```
