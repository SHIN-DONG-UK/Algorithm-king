# 1. 이해

```
- 최소 리모컨 조작 횟수로 출발지에서 목적지까지 이동시킬 수 있음
- 가는 경로 상의 나무를 베고자 함
- N x N 맵에 아빠가 벨 수 있는 나무의 최대 수가 주어짐
- 여기서 목적지까지 이동시키기 위한 최소 조작 횟수를 구하고 싶음
- 차윤이는 항상 위를 바라보는 상태로 RC카 조작을 시작함
```

### 예시

```
GGTGG
GXTTG
GGTTG
GGTYG
GGTGG

'G' : RC카가 이동 가능한 땅
'T' : 나무
'X' : 시작
'Y' : 목적지
```

### (0,2) 나무를 베는 경우 ⇒ 11번

```
G   G > . > G > G
    ^           v
G   X   T   T   G
                v
G   G   T   T   G
                v
G   G   T   Y < G

G   G   T   G   G
```

### (3,2) 나무를 베는 경우 ⇒ 7번

```
G   G   T   G   G

G   X   T   T   G
    v
G   G   T   T   G
    v
G   G > . > Y   G

G   G   T   G   G
```

### 입력

```
- 테스트 케이스 T(MAX 10)
- 맵 크기 N(MAX 10)
- 나무 벨 수 있는 횟수 K(MAX 5)
- 맵 정보
```

### 출력

최소 조작으로 목적지까지 갈 때의 조작 횟수를 출력하라

# 2. 설계

```
[최소 조작 횟수]
- 먼저 최소 조작 횟수를 구하는 방법을 생각해보자
- 최단 거리가 아님을 유의해야 한다
- 제자리 회전도 깊이(lev) 들어가도록 설계하면 된다
- 이렇게 해서 'Y'에 도달했을 때 lev을 업데이트 하면 된다

[나무를 베는 방법]
- 나무를 어떻게 베면 좋을까?
- 사실 모든 경우(모든 위치)를 다 베어보아도 상관없다
- 시간 복잡도가 충분하기 때문이다
- 그럼 나무를 최대 2번 벨 수 있다고 하자
- 일부의 나무를 베어야만 정답이 나오는 경우가 있을까?
- 굳이 2번 벨 수 있는데 1번 벨 필요가 있을까?
- 일단 나무를 최대한 다 베는 편이 더 많은 경우의 수를 탐색할 수 있다
- 그래서 최대 2번이라고 하면 그냥 2번 베자
- 최대 K번 벨 수 있다고 하면, 맵에서 2개를 선택해서 베는 모든 맵을 고려하자
- 조합을 구하는 건 재귀로 구하면 됨
- 그럼 나무를 선택하고, 선택 했을 때의 최소 조작 횟수를 구하면 된다
```
