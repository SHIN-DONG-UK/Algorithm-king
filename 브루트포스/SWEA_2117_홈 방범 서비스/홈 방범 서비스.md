# 1. 이해

```
[상황]
- N x N 맵
- 홈 방범 서비스 -> 마름모 모양만 제공
	- 정사각형 1칸 -> k = 1
	- 정사각형 3칸 -> K = 2
	- 정사각형 5칸 -> K = 3
	- ...
- 운영 비용
	- K * K + (K-1) * (K-1)
	- 도시를 벗어난 영역에 서비스를 제공해도 운영 비용은 변경되지 않음

- 홈 방범 서비스를 제공받는 집들은 각각 M 비용을 지불할 수 있음
- 보안회사는 손해를 보지 않는 한 최대한 많은 집에 홈 방범 서비스 제공

[출력]
- 손해를 보지 않으면서 홈 방범 서비스를 가장 많은 집들에 제공하는 서비스 영역을 찾고,
그 때의 홈 방범 서비스를 제공 받는 집들의 수를 출력하라

[입력]
- Test Case T
- 맵 크기 N ( max 20 ), 지불 비용 M ( max 10 )
- 맵 정보
	1 : 집
	0 : 빈곳
```

# 2. 설계

```
[알고리즘]
- 언제 최대가 될 지 모른다 -> Brute Force 알고리즘
	- 각 칸을 홈 방범 서비스 중심 점으로 시작하여 모든 경우를 계산해본다
	- 각 칸에서 2K-1이 N보다 작거나 같을 때까지 K번 반복해서 모든 경우의 수를 계산한다

[뼈대]
- 2중 for문으로 모든 맵을 돈다
	- 각 칸에서 K크기의 홈 방범 서비스를 만들고 비용을 계산해본다
	- 매번 그 비용을 업데이트 한다

[마름모 만들기]
- 2k-1만큼 돌면서 중심점에서의 거리가 <= k-1 인 점들만 마름모 범위에 들어감
- k제한은 마름모 대각선이 (20, 20)에 도달할 수 있는 범위까지 돌려봐야함
	-> 20root(2) -> 약 30
- 30까지 돌린다
```
