# 1. 이해

```
[상황]
- 길이가 N인 수식
	- 숫자 : 0~9
	- 연산자 : +,-,x
	- 연산자 우선순위 동일 == 왼쪽부터 오른쪽으로 연산
- 괄호 추가
	- 괄호 안에는 하나의 연산자만 존재해야 함

[출력]
- 수식이 주어졌을 때, 괄호를 적절히 추가해 만들 수 있는 식 결과의 최댓값을 구하라

[입력]
- 수식의 길이 N (max 19) -> 2^19 == 백만정도
- N은 홀수
- 첫 번째 입력은 무조건 정수
```

# 2. 설계

```
[알고리즘]
1. 일단 모든 괄호를 씌워본다
2. 씌운 다음 식의 값을 계산한다
3. 최댓값을 갱신한다
=> Brute Force로 모든 경우를 탐색한다
=> 연산자에 표시를 해놓으면, 해당 연산자는 괄호안에 들어가 있는 것이다
=> 현재 연산자를 괄호 씌웠다면, 다다음 연산자부터 시작해야한다

[구현]
- 괄호가 씌워졌다는 표시가 있는 부분부터 계산을 쭉 하고, 나머지를 계산한다
```
