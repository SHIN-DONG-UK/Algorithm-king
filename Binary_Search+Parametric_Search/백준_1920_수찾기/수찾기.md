# 1. 이해

```
1. N개의 정수가 주어진다
	- A[1], A[2], A[N]
	- 1 ≤ N ≤ 100,000
	
2. M개의 정수가 주어진다
	- 1 ≤ M ≤ 100,000

M개의 정수가 각각 A안에 존제하는지 검사하면 된다
```

# 2. 설계

```
1차 for문으로 탐색한다고 하자
최악의 경우, 100,000 x 100,000번 연산을 수행해야 함
-> 백억 -> 시간초과

이 문제는 전형적인 Binary Search 문제임
Binary Search를 하면 범위가 십만이니까 하나를 찾는데 17번만 하면 됨

M개의 정수를 A에서 존재하는지 찾으면 되잖아?
그럼 A가 정렬되어 있는게 좋겠지?
```
