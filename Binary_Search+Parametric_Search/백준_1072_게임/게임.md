# 1. 이해

```
[상황]
형택이는 이제 게임에서 지지 않음 -> +1판 == +1승

[게임 기록]
게임 횟수 : X
이긴 게임 : Y(Z%)
Z는 형택이의 승률, 소수점 버림

[출력]
X와 Y가 주어졌을 때, 형택이가 최소 몇 번 더 해야 Z가 변하는지?
Z가 절대 변하지 않으면 -1을 출력
```

# 2. 설계

```
[제한]
1 <= X <= 1,000,000,000
0 <= Y <= X

[생각]
- Z의 수식을 생각해보자
	Z = Y/X * 100
	만약 게임 횟수가 n이라고 하면,
	Z = (Y+n/X+n) * 100
- X와 Y가 같지 않은 이상, n이 매우 크면 무조건 Z가 바뀜
- 그런데 언제 바뀌는지 알아야 하니까, 하나씩 늘려가면서 그 값을 찾으면 되지 않을까?
	하나씩 늘리면서 찾으면 무조건 찾을 수 있긴 함.
	근데 시간초과 날 수 있음.
	그래서 그 지점을 Binary Search로 찾으면 됨

[Binary Search]
- 퍼센티지가 바뀌면 왼쪽으로 가고. 안바뀌면 오른쪽으로 가면 됨
- 그래서 start <= end인 지점 찾으면 됨
- start = 0
	end = 1000000000
	아 잠깐잠깐
	99인 애들은 바뀔 수 없음
```

# 3. 주의

```
// y/x*100와 (y*100)/x는 다름!
#include <iostream>
using namespace std;

int main() {
	float num = 0.0;
	for (int i = 0; i < 100000; i++)
	{
		num += 0.1;
	}

	cout << "예상 = 10000\n";
	cout << "실제 = " << num << '\n';
}
// 예상 = 10000
// 실제 = 9998.56
```
