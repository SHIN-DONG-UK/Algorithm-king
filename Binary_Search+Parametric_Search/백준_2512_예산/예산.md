# 1. 이해

```
- 예산 정해짐
- 요청이 들어옴
- 배정 방법
1. 모든 요청이 배정될 수 있는 경우에는 요청한 금액을 그대로 배정
2. 모든 요청이 배정될 수 없는 경우에는 특정한 정수 상한액을 계산하여 그 이상인 예산요청에는
모두 상한액을 배정한다. 상한액 이하의 예산요청에 대해서는 요청한 금액을 그대로 배정한다.

[예시]
국가 예산 : 485
예산 요청 : 120 110 140 150 (총 520)
상한액 : 127
배정된 예산 : 120 110 127 127 (총 484)
```

# 2. 설계

```
[제한]
1. N
	3 <= N <= 10,000
	각 예산은 1이상 100,000이하 -> 모든 예산의 총 합이 10억을 넘지 않음 -> int형으로 sum
2. M
	N <= M <= 1,000,000,000
	int 형으로 start와 end를 설정해도 된다

[생각해보기]
- 상한액의 최댓값을 찾아야 한다
- 나무 자르는 문제랑 비슷함
- 상한액의 start와 end를 어디를 설정해야 할까?
	start = 0
	end = 요청 예산 중 최댓값+1
- mid = (start + end) / 2 하고, 만약 해당 상한액이 예산을 만족하면 기록하고 오른쪽
	만족하지 않으면 왼쪽 == Parametric Search
```
