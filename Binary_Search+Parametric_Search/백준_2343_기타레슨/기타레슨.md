# 1. 이해

```
[블루레이]
i번 강의와 j번 강의를 같은 블루레이에 녹화하려면 i와 j 사이의 모든
강의도 같은 블루레이에 녹화해야 함

[입력]
- N개의 강의 (1 <= N <= 100,000)
- M개의 블루레이 (1 <= M <= N)
- 각 강의는 1 <= t <= 10000
- 강의 순서대로 입력됨
- 그런데 오름차순인 것은 보장되지 않음(애초에 이 문제는 정렬과 무관함)

[출력]
각 블루레이의 크기는 같아야 함
이때 블루레이 크기의 최솟값을 구하라
```

# 2. 설계

```
[설계]
- M개의 블루레이로 각 영상을 쪼개야 함
- 예를 들어 
9 3
1 2 3 4 5 6 7 8 9
라고 주어지고
12345 67 89 로 쪼개면
15 13 17이 되어 블루레이의 크기는 17이 된다
이보다 더 작은 블루레이를 만들 수 없다

[생각]
- 블루레이 크기를 정하고 등분하면 되지 않을까?
	그래서 가능 여부를 따지면 되잖아
	가능 여부를 어떻게 판단하지?
	-> 그리디하게 최대한 강의를 블루레이에 담고, 담긴 블루레이 개수가 M제한보다 작다면 그 크기는 가능한 것
	
- Binary Search로 블루레이 크기 정해주면 됨

[Binary Search]
- M == 3일 때,
	3등분보다 크다 -> 오른쪽
	3등분보다 작다 -> 기록하고 왼쪽
- start = 0
	end = sum(강의들) -> sum(강의들)보다 클 수 없다
```

# 3. 주의

```
1. start = 1로 시작해야 한다
	강의 길이는 자연수이므로 0이 될 수 없다
	
2. 강의 하나가 블루레이 크기보다 큰 경우, 해당 케이스는 성립하지 않는다
	-> check함수에서 이 케이스에 false를 리턴할 수 있어야 함
```
