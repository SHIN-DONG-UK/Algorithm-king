## 💡 BOJ 5557 - 1학년 [[문제 링크](https://www.acmicpc.net/problem/5557)]

| 항목 | 내용 |
|------|------|
| 문제 유형 | `DP` |
| 사용 알고리즘 | |
| 난이도 | `골드5` |

### ✅ 문제 설명
**[문제 요약]**

- 상근이는 숫자 배열에서 마지막 두 숫자 사이에는 '='을 넣고, 나머지 숫자 사이에는 '+' 또는 '-'를 넣어 등식을 만들고 있다
- 상근이는 올바른 등식을 만들려고 한다
- 상근이는 음수를 모른다
- 상근이는 20이 넘는 수를 모른다
- 따라서 중간에 나오는 수가 모두 0이상 20이하여야 한다
- 숫자가 주어졌을 때, 상근이가 만들 수 있는 올바른 등식의 수를 구하라

**[제약 사항]**

- 숫자의 개수 3 <= N <= 100
- 주어지는 숫자는 0~9
- 등식의 개수는 2^63 - 1 이하

### 🔍 풀이 접근
- 일단 DFS 탐색과 가지치기는 TLE다 
- 중간 계산 결과는 항상 0 이상 20 이하여야 한다
- 경우의 수를 누적하는 DP 문제로 보인다
- 상태를 어떻게 잡을까?

### 💡 핵심 로직
- dp[i][v]
  -> i번째 숫자까지 사용했을 때,
  -> 중간 결과가 v가 되는 경우의 수
-이렇게 하면, 다음 로직이 가능해진다
  - i번째 숫자를 더하거나 뺄건데, 0이상 20이하의 중간결과 중 유효(>0)한 중간결과에 더하거나 뺀다
   - 이렇게 N-1까지 누적한 후, N번째 숫자와 같은 숫자의 경우의 수를 정답으로 출력한다

### 📌 배운 점
- 2차원 DP 문제의 새로운 유형의 문제를 풀이했습니다.
- i번째 요소에 대해 가능한 모든 숫자 테이블을 만들어두고, 이 테이블을 다 뒤져서 누적해나가는 문제를 이해했습니다.
