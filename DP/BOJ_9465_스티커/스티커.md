# 1. 이해

```
[스티커]
- 2행 n열로 배치
- 스티커 한 장을 뗴면, 상하좌우로 붙은 스티커 같이 떼짐
- 점수의 합이 최대가 되게 스티커를 떼어내려고 함

[출력]
- 스티커 떼서 얻을 수 있는 점수 최댓값을 구하라

[입력]
- 테스트 케이스 T
- n (1 ≤ n ≤ 100,000)
- 점수들
```

# 2. 설계

```
[직접 해보기]
(case1)
50 10 100 20 40
30 50 70 10 60

왼쪽부터 스티커를 뗴보자
D[k]에는 k번째 스티커까지 최적해가 들어있다고 생각

얘는 안뗴는 선택지도 있음
그래서, 선택지가 몇 가지 있냐면
1. 안뗸다
2. i번째 열의 0행을 뗀다
3. i번째 열의 1행을 뗀다

이 선택지들 중에 가장 큰 값을 D에 저장해 놓으면 항상 최적해를 구할 수 있다고 생각함

D[0][i] = max(D[0][i-2], D[1][i-2], D[1][i-1]) + A[0][i];
D[1][i] = max(D[0][i-2], D[1][i-2], D[0][i-1]) + A[1][i];

이렇게 구해나가면 최댓값은 어디에 기록되어 있는가?
마지막 열의 0행과 1행 중에 최댓값이 있겠죠?
```
