## 💡 BOJ 10835 - 카드게임 [[문제 링크](https://www.acmicpc.net/problem/10835)]

| 항목 | 내용 |
|------|------|
| 문제 유형 | `DP` |
| 사용 알고리즘 | `메모이제이션` |
| 난이도 | `골드5` |

### ✅ 문제 설명

**[문제 요약]**

- 카드가 있는데, 각 카드에는 양의 정수 하나가 적혀있다
- 같은 숫자가 적힌 카드가 여러 장 있을 수 있다
- 게임은 다음과 같다
- 짝수개의 카드를 섞어 두 더미로 나누어 왼쪽과 오른쪽에 둔다
- 언제든지 왼쪽 카드만 통에 버릴 수도 있고 왼쪽 카드와 오른쪽 카드 둘 다 통에 버릴 수도 있는데, 이때 얻는 점수는 없다
- 오른쪽 카드에 적힌 수가 왼쪽 카드에 적힌 수보다 작은 경우에는 오른쪽 카드만 통에 버릴 수도 있는데, 오른쪽 수만큼 점수를 얻는다
- 게임을 진행하다가 어느 쪽 더미든 남은 카드가 없다면 게임이 끝나며 그떄까지 얻은 점수의 합이 최종 점수다
- 두 더미의 카드가 주어졌을 때, 얻을 수 있는 점수의 최댓값을 출력하라

**[제약 사항]**

- 한 더미의 카드 수 N-> 1 <= N <= 2,000
- 정수 범위 1 ~ 2,000

### 🔍 풀이 접근

- 세 가지 선택지를 N번 반복하면 모든 경우를 고려할 수 있다
- 단, 불가능한 경우는 가지치기한다
- 그런데, 문제의 크기가 2,000이므로 TLE다
- 그리디하게 문제를 풀어낼 수 있지 않나? -> 반례 존재
- DP로 풀어보자

### 💡 핵심 로직

- 왼쪽 더미를 행으로, 오른쪽 더미를 열로 하여 2차원 DP를 구성한다.
- 그리디하게 현재 왼쪽 카드가 오른쪽 카드를 먹을 수 있으면, 계속 먹는다.
- 만약 현재 왼쪽 카드가 오른쪽 카드를 먹을 수 없으면, 재귀적으로 아래쪽으로 이동하거나 오른쪽 대각선으로 이동한다.
- 위 2번과 3번을 반복한다.
- 만약 현재 dp가 계산된적 있으면, 최댓값이 보장되므로 그걸 리턴한다.

### 📌 배운 점
- 후위 순회를 통해 dp가 계산된 적 있으면 최댓값이 보장된다는 것을 알게되었습니다.
