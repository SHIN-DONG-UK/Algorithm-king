# 1. 이해

```
[상황]
- 맵 : N x M (1 ≤ N,M ≤ 1000)
- 1 : 벽
- 0 : 이동 가능
- (1, 1) -> (N, M)로 최단경로 이동
- 벽 한 개는 지나갈 수 있음

[출력]
- 최단 거리를 출력하라
- 이동 불가능하면 -1 출력

[입력]
- N, M
- 맵
```

# 2. 설계

```
1. 일단 벽 부수기를 고려하지 않으면 플러드필인데 조건달아서 가는거임
2. 이제 벽을 부수는 걸 고려해보자

그냥 노드에다가 벽 부순 기록 달아두면 되는거 아니노?
그래서 벽을 부순 노드로부터 연결된 노드들은 이제 기록을 받고 더 이상 벽을 부수지 못함
벽 안부순 노드로부터 연결된 노드들은 벽 부술 수 있음

- 이 문제는 벽을 부수고 이동한 노드가 먼저 도착했을 때 문제가 발생함
- 먼저 벽 뚫고 갔으니 방문처리를 했다. 근데 더 이상 못간다. 그럼 벽 안뚫고 간 경로가 뚫어줘야 하는데
방문처리를 해버려서 그냥 끝나버린거지
- 그럼 뭐다? 경로를 분리해줘야 함
- 벽을 뚫고 지나온 경로랑 벽을 안뚫고 지나온 경로를 분리해서 플러드 필하면 됨
- 그럼 항상 최단거리가 기록되면서 경로를 분리했기 때문에 항상 최적해를 구할 수 있음

- 정답을 출력할 때는 두 배열의 (N, M) 좌표값을 확인
	1. 둘 다 도달 못했다 -> -1
	2. 하나가 도달 못했다 -> 도달한거 출력
	3. 둘 다 도달했다 -> 둘 중 작은 값 출력
```
