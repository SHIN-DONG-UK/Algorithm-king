## 💡 BOJ 2668 - 숫자고르기 [[문제 링크](https://www.acmicpc.net/problem/2668)]

| 항목 | 내용 |
|------|------|
| 문제 유형 | `그래프 탐색` |
| 사용 알고리즘 | `DFS` |
| 난이도 | `골드5` |

### ✅ 문제 설명
**[문제 요약]**

- 세로 두 줄, 가로 N개의 칸으로 이루어진 표가 있다
- 첫째 줄의 각 칸에는 1~N이 차례대로 들어 있다
- 둘째 줄의 각 칸에는 1이상 N이하인 정수가 들어있다
- 첫째 줄에서 숫자를 적절히 뽑으면, 그 뽑힌 정수들이 이루는 집합과, 뽑힌 정수들의 바로 밑의 수의 집합은 일치
- 이러한 조건을 만족시키도록 정수들을 뽑되, 최대로 많이 뽑는 방법을 구하라

**[제약 사항]**

- 1 <= N <= 100

### 🔍 풀이 접근
- 주어진 표를 Edge라고 생각해보자
- 첫째 줄 -> 둘째 줄의 관계를 Graph로 표현하고 여기서 모든 싸이클을 찾고 그 크기를 더하면 되지 않을까?
- 첫째 줄을 from이라 하고 둘째 줄을 to라고 하면 문제 요구사항과 동치가 된다

### 💡 핵심 로직
- 표를 보고 방향 Graph를 만들어둔다
- 1번 노드부터 N번 노드까지 모든 노드에서 DFS 탐색을 수행해본다
- DFS 탐색
  - 다음 노드로 이동하는데, 만약 시작 노드로 돌아왔으면 사이클로 판정한다
  - 다음 노드가 이미 방문했으면, 이미 판정이 끝난 것이므로 더 이상 DFS 탐색을 수행하지 않는다

### 📌 배운 점
- 방향이 있는 그래프에서 사이클을 탐색하는 방법을 공부했습니다.
- 문제 요구사항을 그래프로 추상화하는 연습을 했습니다.
