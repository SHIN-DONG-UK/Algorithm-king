# 1. 이해

```
[소개팅]
- 임의의 두 지렁이를 매칭한 후 다른 지렁이가 있는 곳으로 가도록
- 가능한 움직인 벡터 합의 크기가 작기를 바람

[지렁이 이동]
- 지렁이 A->B == (x,y) => 크기 x*x+y*y

[출력]
모든 지렁이들을 매칭시키고 소개팅을 주선하되, 각 지렁이들이 움직인 벡터 합의 최솟값
```

# 2. 설계

```
- N=10이라고 하자
- 모든 경우를 따져보기 위해, 20C2 -> 18C2 -> ...
- 이렇게 할 필요가 없다
- start를 정하기만 하면, start인 지렁이는 다 빼고 end인 지렁이는 다 더하면 되기 때문이다.
- 즉, 지렁이 그룹을 선택하는 순서는 상관없으며, 시작 지렁이 절반을 정하기만 하면
모든 경우의 수를 따져볼 수 있다

[움직인 벡터 합]
- N개중 N/2개를 선택하는 경우의 수를 구한댜(nCn/2)
- 해당 index를 visited에 기록하여 1이면 start로하고, 0이면 end로 한다
- 기저 조건에 도달하면, 벡터의 합을 구하고 최솟값을 갱신한다

[주의]
- 지렁이의 위치는 y,x 각각 -100000~100000이 가능하다
- 최악의 경우, 벡터의 합은 (100000)^2 + (100000)^2이다
- 따라서, 800억의 수를 담을 수 있는 long long 타입으로 변수를 선언해야 한다
```
