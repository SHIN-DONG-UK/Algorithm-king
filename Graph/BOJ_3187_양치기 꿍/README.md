## 💡 BOJ 3187 - 양치기 꿍 [[문제 링크](https://www.acmicpc.net/problem/3187)]

| 항목 | 내용 |
|------|------|
| 문제 유형 | `그래프 탐색` |
| 사용 알고리즘 | `BFS` |
| 난이도 | `실버1` |

### ✅ 문제 설명
**[문제 요약]**

- 울타리 안에 양의 숫자 > 늑대 숫자 => 양이 늑대를 잡아 먹음
- 울타리 안에 양의 숫자 <= 늑대 숫자 => 늑대가 양 모두 잡아먹음
- 2차원 맵이 주어질 때, 살아남는 양과 늑대의 수를 구하라

**[제약 사항]**

- 양과 늑대는 대각선으로 이동할 수 없다
- 2차원 맵의 가로, 세로 길이 <= 250

### 🔍 풀이 접근
- BFS 문제이다
- 다만, 2차원 맵을 모두 훑으면서 BFS를 탐색하는 Brute-Force 형식의 BFS라고 할 수 있다

### 💡 핵심 로직
- 2차원 격자를 모두 탐색한다
- 만약 현재 위치가 방문하지 않았고, 벽이 아니라면 BFS 탐색을 수행한다
   - BFS 탐색은 인접한 노드로 이동하면서 양과 늑대의 개수를 센다
   - 벽을 만나면 더 이상 큐에 집어넣지 않는다
   - 큐가 비었으면 루프를 나와서, 양의 개수와 늑대의 개수를 비교한다
   - 양의 개수 > 늑대 개수: 전체 양 개수에 양을 합친다
   - 양의 개수 <= 늑대 개수: 전체 늑대 개수에 늑대를 합친다

### 📌 배운 점
- 전형적인 BFS 탐색 문제를 풀이했습니다.
