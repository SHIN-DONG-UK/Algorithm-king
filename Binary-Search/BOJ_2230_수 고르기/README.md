## 💡 BOJ 2230 - 수 고르기 [[문제 링크](https://www.acmicpc.net/problem/2230)]

| 항목 | 내용 |
|------|------|
| 문제 유형 | `투 포인터` |
| 사용 알고리즘 | `이분 탐색` |
| 난이도 | `골드5` |

### ✅ 문제 설명
**[문제 요약]**

- N개의 정수로 이루어진 수열 A[1], A[2], ... , A[N]이 있다
- 이 수열에서 두 수를 골랐을 때(같은 수 일 수 있다) 그 차이가 M이상이면서 제일 작은 경우를 구하라

**[제약 사항]**

- N <= 100,000
- M <= 20억
- -10억 <= A[i] <= 10억

### 🔍 풀이 접근
- O(N^2)의 복잡도로는 TLE이다
- 이분탐색 그냥 갈기면 되는거 아님?
- 두 수 (a, b)를 찾는다고 할 때, a는 N개의 수를 다 해보고, b는 이분탐색으로 찾으면 되잖슴

### 💡 핵심 로직
- for loop을 i=0부터 i=N-2까지 돈다
- left = i + 1 / right = N-1로 해서 이분탐색을 수행한다
- M이상이면 왼쪽 범위를 탐색하고, 그렇지 않으면 오른쪽 범위를 탐색한다 -> 가장 마지막에 찾은 답이 현재의 최솟값임
- 이들 중 가장 작은 값을 찾으면 됨
