## 💡 BOJ 17503 - 맥주 축제 [[문제 링크](https://www.acmicpc.net/problem/17503)]

| 항목 | 내용 |
|------|------|
| 문제 유형 | `Binary-Search` |
| 사용 알고리즘 | `그리디`, `정렬` |
| 난이도 | `실버1` |

### ✅ 문제 설명
**[문제 요약]**

- N일 동안 맥주 축제가 열린다
- K종류의 맥주가 제공된다
- 참가자는 하루에 맥주 1병만 받을 수 있고, 이전에 받았던 맥주 종류는 다시 받을 수 없다
- K종류의 맥주에 각각 '선호도'와 '도수 레벨'이 존재
- N일동안 N개의 맥주를 마시려고 한다
- 맥주 N개의 선호도 합이 M이상 되려고 한다
- 마시는 맥주의 도수 레벨이 간 레벨보다 높으면 안된다
- 조건을 만족하는 간 레벨의 최솟값을 구하라

**[제약 사항]**

- N <= 200,000
- M < 2^31
- N <= K <= 200,000
- 선호도 <= 10,000
- 도수 레벨 < 2^31

### 🔍 풀이 접근
- 총 K개의 맥주가 있고 이 중 N개를 선택해야 한다
- K개 중에 N개를 선택하는 모든 경우의 수를 구해서 푸는 건 말이 안된다
- 도수 레벨을 정해놓고, 이게 가능한지 아닌지 체크하는 방식으로 문제를 풀면 되지 않을까? == Parametric Search

### 💡 핵심 로직
- 선호도가 큰 순서대로 정렬해 놓는다
- 도수 레벨을 이분탐색으로 try해보면서, 최솟값을 갱신한다

`check(int level)`
- 선호도를 처음부터 탐색하면서, level이하인 맥주들을 N개 더한다
- 만약 더한 값이 M이상이면 `true` 그렇지 않으면 `false`를 리턴

`시간복잡도`
- check함수 -> 1번에 K
- 이분탐색 -> 31번
- 총 : 31 * K
	
### 📌 배운 점
- 큰 틀을 이분 탐색으로 하고, 현재 `mid`에서 `check`하는 함수에서 정렬과 그리디한 방식을 접목하는 문제를 풀었습니다.
