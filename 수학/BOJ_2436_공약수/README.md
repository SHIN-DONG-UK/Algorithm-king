## 💡 BOJ 2436 - 공약수[[문제 링크](https://www.acmicpc.net/problem/2436)]

| 항목 | 내용 |
|------|------|
| 문제 유형 | `수학` |
| 사용 알고리즘 | |
| 난이도 | `골드5` |

### ✅ 문제 설명
**[문제 요약]**

- 두 개의 자연수가 주어졌을 때, 이 두 수를 최대공약수와 최소공배수로 하는 두 개의 자연수를 구하라

**[제약 사항]**

- 두 자연수는 2이상 1억이하
- 두 개의 자연수가 여러 개인 경우, 두 수의 합이 최소가 되는 수를 출력하라

### 🔍 풀이 접근

- 최대공약수는 공통 약수 중 가장 큰 값을 구하는거임
- 최소공배수는 최대공약수에 각 몫을 곱하면 됨
- 최소공배수를 최대공약수로 나눈 몫이 되도록 하는 두 수를 구해보자
- 6 180은 30이니까 30이 되는 두 개의 수는 구하기 쉽다
- 2 100000000은 5천만 ->곱해서 5천만이 되는 두 개의 수를 구하면 된다
- 곱해서 5천만이 되는 두 개의 수는 하나를 정하면 나머지 하나가 결정된다
- 그래서 최악의 경우 5천만번만 돌면 된다
- 그런데 sqrt(5천만)이후로는 중복되므로, sqrt(N)이 최종 시간복잡도가 된다
- 단, 두 수는 서로소여야 한다
- 그래야 A가 최대공약수가 됨

### 💡 핵심 로직

- 최소공배수를 최대공약수로 나눈다
- 나눈 몫을 tar라고 할 때, sqrt(tar)를 돌면서 tar가 나누어 떨어지는 수를 찾는다
- 단, 두 수는 서로소여야 한다
- 나누어 떨어지는 두 수 중 합이 가장 작은 값을 업데이트한다

### 📌 배운 점
- gcd 알고리즘에 대해 다시 한 번 상기했습니다.
- 두 수의 곱에 관한 관계를 구할 때, 하나의 수를 결정하면 다른 수가 자동으로 결정되어
시간복잡도가 O(N^2)이 아니라 O(N)임을 상기했습니다.
