## 💡 BOJ 1456 - 거의 소수 [[문제 링크](https://www.acmicpc.net/problem/1456)]

| 항목 | 내용 |
|------|------|
| 문제 유형 | `수학` |
| 사용 알고리즘 | `에라토스테네스의 채` |
| 난이도 | `골드5` |

### ✅ 문제 설명
**[문제 요약]**

- 어떤 수가 소수의 N제곱(N >= 2)꼴일 때, 그 수를 거의 소수라고 한다
- 두 정수 A와 B가 주어지면, A보다 크거나 같고, B보다 작거나 같은 거의 소수가 몇 개인지 출력하라

**[제약 사항]**

- 1 <= A <= B <= 10^14

### 🔍 풀이 접근
- 1 10일 때 한 번 해보자
- 4 <- 2^2
- 8 <- 2^3
- 9 <- 3^2
- sequential하게 B보다 작거나 같은 정수를 찾는다고 해보자
- 최악의 경우는 B=10^14일 때, 2의 거듭제곱이 가능한 개수를 찾는 것이다
- 2^10 ~= 10^3 -> (2^10)^4 -> 2^40
- unsigned long long으로 충분한데?
- O(1)이라고 봐도 무방하다
- 소수만 잘 구해놓고 sequential하게 찾아도 될 것같다

### 💡 핵심 로직
- 필요한 소수의 범위는 1부터 sqrt(B)이다 -> worst case == 10^7
- 에라토스테네스의 채를 활용해 소수인 수를 벡터에 넣는다
- 벡터를 돌면서 sequential하게 개수를 구한다

### 📌 배운 점
- overflow에 대해 고민하는 시간을 가졌습니다.
- overflow를 방지하기 위해 bound를 제곱수로 나눈다는 생각을 할 수 있었습니다.
