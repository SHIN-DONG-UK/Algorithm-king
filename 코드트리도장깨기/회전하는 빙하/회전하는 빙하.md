# 1. 이해

```
[격자]
- 2^n x 2^n
- 빙하와 얼음 -> 회전 인접성
- 좌측 상단에 있는 빙하를 특정 범위를 선택해서 회전
	-> 다른 상하좌우 인접한 격자들도 똑같은 크기만큼 회전
	-> 빙하를 회전하는 범위를 '레벨'
	-> 레벨이 L일 때 2^Lx2^L만큼 격자를 선택하여 2^L-1x2^L-1만큼 잘라
	4등분하여 회전
	
	
[녹음]
- 한 칸을 기준으로 상하좌우 인접한 칸에 얼음이 3개 이상 이면 안녹음
- 만약 0(없음)이 두 개이상이라면 1이 줄어듦
- 격자를 벗어나는 경우도 얼음이 없는 것
- 녹는 것은 동시에 진행 -> next_map

[출력]
- 모든 회전을 끝내고 난 뒤 남아있는 빙하의 총 양과 가장 큰 얼음 군집
의 크기를 출력하라

[입력]
- 회전 가능 레벨 n과 회전 횟수 q가 주어짐
- 회전 가능 레벨이 n이라는 의미는, 2^n x 2^n의 크기라는 뜻
- 맵 정보
- 회전 레벨이 q개 주어짐
```

# 2. 설계

```
[설계]
1. 회전 레벨을 입력받아 회전을 하는 함수가 필요
	-> 예술성 문제에서 한 것처럼 기준 좌표와 배열 크기로 회전 가능
	-> 먼저 기준 좌표를 가리켜보자
2. 이 함수가 있다면 Simulation() 함수 안에서 회전을 반복 수행하면 됨
3. 회전을 모두 수행했다면, 빙하의 총양을 계산
	-> 2중 for문으로 총양 계산
4. 또한 BFS 수행해서 가장 큰 얼음 군집 탐색
	-> 얼음 문제
```
