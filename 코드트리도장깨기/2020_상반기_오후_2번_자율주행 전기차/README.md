# 1. 이해

```
[문제 요약]

- n x n 격자 도로가 있음
- 벽의 위치와 m명의 승객 위치가 주어질 때, 주어진 배터리로 승객을 모두 태울 수 있는지 알아보고자 함
- 승객을 태우러 갈 때나 목적지로 이동할 때 항상 최단거리로 움직임
- 한 칸 이동할 때 배터리 1 소모
- 승객을 목적지에 태워주면 그 승객을 태워서 이동한 배터리의 두 배를 충전
- 이동 중에 배터리가 모두 소모되면 그 즉시 종료
- 승객이 여러 명일 경우, 현재 위치에서 최단 거리가 가장 짧은 승객을 먼저 태움
    - 그런 승객이 여러 명일 경우 가장 위에 있는 승객
      - 또 여러 명이면 가장 왼쪽에 있는 승객
- 도로의 상태가 주어질 때 모든 승객을 성공적으로 데려다 줄 수 있는지 알아내고, 데려다 줄 수 있는 경우 최종 배터리를 출력
- 데려다줄 수 없으면 -1 출력

[제약 사항]

- 격자 크기 n <= 20
- 승객 수 m <= n^2
- 초기 배터리 충전량 c <= 500,000
- 모든 승객의 출발지와 도착지는 빈칸
- 각 승객의 출발지와 목적지는 모두 다름
- 단, 한 승객의 출발지와 다른 승객의 목적지는 같을 수 있음
- 서로 다른 승객의 출발지가 동일한 경우는 없다
```

# 2. 설계

```
[나이브한 생각]

- 일단 출발지는 서로 다르니까, 출발지를 맵에 표시를 해야 한다
- 그래야 거리 정보를 얻을 수 있다
- 각 승객에게 ID를 부여하고, ID를 맵에 표시하자
- 그리고 ID를 통해 도착점을 조회할 수 있도록 하면 된다
- 맵에서 벽을 -1로 바꾸고, ID를 1부터 부여하자
- 현재 택시 위치에서 가장 가까운 승객을 찾는다
    - 상좌하우 순서로 BFS 탐색한 다음, 최초로 만나는 승객 ID에서 바로 리턴때려도 된다
    - 상좌하우 순서로 BFS 탐색하면 같은 거리에 있는 대상에 대해 우선순위가 적용되게 하는 것

- 도착지에서 배터리가 음수이면 리턴하면 되고, 0이상이면 충전하면 된다

[설계]

- n x n 격자에서 벽은 -1로, 출발지는 승객의 ID를 표시한다

//의사 코드
while(1){
- 택시의 현재 위치에서 BFS 탐색을 수행해 최단거리에 있는 승객을 태우러 간다
    - 단, 현재 위치에 승객이 있는 경우도 체크한다
- 그리고 도착점에 태워준다
    - 도착점에 도달했는데 배터리가 음수이면 리턴한다
- 이동거리의 두 배만큼 충전한다
}
```
