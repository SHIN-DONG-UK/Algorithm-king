[문제 요약]

- 해적 선장 코디는 함대를 이끌고 대형 선박을 침몰시키려고 한다
- 코디는 총 T개의 명령을 순차적으로 내린다
- 각 명령은 아래 네 가지 유형 중 하나다

```
1. 공격 준비

- 코디가 대형 선박을 공격하기 위해 N척의 선박에 사격 준비를 지시
- 각 선박은 고유 id, 공격력 p, 재장전 시간 r을 가지며, 초기 상태는 모두 사격 대기 상태
- 100 N id1 p1 r1 id2 p2 r2 ... idN pN rN 형식으로 입력
- 이 명령은 첫 줄 이후에 단 한번만, 그리고 반드시 주어지며, 선박 번호 id는 중복x


2. 지원 요청

- 추가 병력을 요청하여 새로운 선박이 합류
- 새로 합류한 선박은 사격 대기 상태로 추가되며, 선박 번호 id, 공격력 p, 재장전 시간 r을 가짐
- 200 id p r 형식으로 입력
- 이미 존재하는 선박 번호 id가 입력으로 주어지지 않는다


3. 함포 교체

- id번 선박의 함포를 교체함
- 교체 후 해당 선박의 공격력은 pw가 됨
- 300 id pw 형식으로 입력
- id번 선박의 공격력을 pw로 갱신
- 존재하지 않는 id가 주어지지 않는다

4. 공격 명령

- 사격 대기 상태인 선박 중 공격력이 가장 높은 선박 최대 5척에 일제 사격을 명령
- 공격력이 같다면 선박 번호 id가 가장 작은 선박을 우선 선택하며, 총 피해가 최대가 되도록 선박을 고른다
- 사격에 참여한 선박들의 공격력 합만큼 대형 함선에 피해를 줌
- 사격한 선박은 즉시 재장전에 들어가며, 사격 시점을 포함해 r시간이 경과하면 다시 사격 대기 상태로 전환
- 재장전 중인 선박은 공격할 수 없음
- 400 형식으로 입력

```

- 각 명령은 1시간 단위로 실행됨
- i번째 명령이 수행된 뒤 1시간이 지나면, i+1번째 명령이 수행됨
- 공격 명령이 주어질 때마다, 해당 차례의 총 피해량, 사격에 참여한 선박 수, 그리고 사격 우선순위에 따른 사격 선박들의 id를 출력하라

[제약 사항]

- 지원 요청 횟수 <= 30,000
- 함포 교체 <= 10,000
- 공격 명령 <= 5,000
- 명령 횟수 T: 2 <= T <= 50,000
- 초기 선박 개수 N: 1 <= N <= 10,000
- id <= 10억
- p, pw <= 1백만
- 재장전 시간 r: 1 <= r <= 10

[나이브한 생각]

- 공격 명령 함수 호출을 염두에 둔 설계가 필요하다
- 왜냐하면 가장 연산이 많은 함수이기 때문
- 매번 공격력이 가장 높은 선박 5개를 선택해야 한다
- 또한, 이 공격력은 매번 업데이트될 수 있다
- 함수를 호출할 때마다 정렬을 수행한다고 하자
- 최악의 경우, 40,000 x log40000 = 600,000
- 정렬 후 5개를 선택할 때 사격 대기 상태인 애들만 골라야 하므로 선박을 고르는데 평균적으로 100번 걸린다고 해보자 -> 6천만
- 애매하지만, 일단 매번 정렬을 수행하는 것으로 설계해보고, 이후 시간복잡도를 줄일 방법을 적용하자

[설계]

<자료구조>

- 선박 구조체
	- id
	- power
	- rTime
	- lastAttack
- 선박 배열 -> 최대 길이 40000 + 1

- id -> 선박 인덱스 hash map
- 시간 변수 t

<알고리즘>

1. init()

- 초기 선박의 개수 N 초기화
- N개의 선박을 선박 배열에 초기화
- 선박은 공격 대기 상태 -> lastAttack을 -10으로 초기화하면 무조건 공격 대기 상태가 됨
- id -> 선박 인덱스 hash map 초기화

2. request(int id, int p, int r)

- 선박 개수를 +1 증가시켜서 새로운 공간에 선박 생성
- 선박은 공격 대기 상태 -> lastAttack을 -10으로 초기화하면 무조건 공격 대기 상태가 됨

3. replacement(int id, int pw)

- id -> 인덱스 map으로 선박 인덱스를 얻는다
- 선박 인덱스로 선박에 접근해 공격력을 업데이트한다

4. attack()

- 선박 배열을 '우선순위'에 따라 정렬한다
- 선박 배열을 돌면서, '최대' 5개의 선박을 선택한다 (cnt < 5)
	- 만약 now_t - lastAttack < r이면, 해당 선박은 재장전 상태
	- 그렇지 않다면, 해당 선박은 공격 대기 상태
	- 공격 대기 상태인 선박들만 골라야 함
	- 이런 선박의 lastAttack = now_t로 업데이트
	- sum += power 합치기
	- cnt += 1
	- vector.push_back(id)

- sum 출력 cnt 출력 vector 출력


[최적화]

- 매번 정렬을 수행할 경우, 시간초과가 발생
- 모든 아이템을 정렬하지 말고, 정렬된 자료구조를 갖고 있는게 시간초과를 막을 방법이다
- 그런데 문제는 정렬 기준이 매번 업데이트 되어 정렬 자료구조를 최신화하기 번거롭다는 것
- 즉, 기존 정렬된 자료구조에서 정렬 기준이 업데이트되면, 이미 존재하는 아이템은 dummy가 됨
- dummy인지 아닌지 판별할 수만 있다면, lazy-deletion하면 됨
- 원본은 항상 임의 접근 가능한 배열에 있으므로, 정렬된 자료구조의 아이템이 더미인지 판단할 수 있음
- 정렬된 자료구조는 heap구조이면 되고, lazy-deletion을 수행하면 되겠다
