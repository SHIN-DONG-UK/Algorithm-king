# 1. 이해

```
[놀이기구]
- N x N

[학생 선호 테이블]
- 학생번호 -> 좋아하는 학생 번호 4명씩
- 좋아하는 학생 번호는 중복되지 않음
- 자기 자신은 없음

[탑승 방법]
1. 격자를 벗어나지 않는 4방향으로 인접한 칸 중 좋아하는 친구 가장 많은 곳으로
2. 1번 조건을 만족하는 칸이 여러 곳이라면, 인접한 칸이 가장 많이 빈 곳으로 이동
단, 맵을 벗어난 곳은 빈칸으로 치지 않음
3. 2번 조건도 여러 곳이라면, 그 중 행 번호가 가장 작은 위치
4. 3번 조건도 여러 곳이라면, 그 중 열 번호가 가장 작은 위치

[점수 산정]
- 각 학생마다 인접한 곳에 좋아하는 학생수 테이블
- 이 테이블에 따른 점수 합산

[출력]
- 들어오는 학생의 순서와 각 학생마다 좋아하는 4명의 학생 번호가 주어졌을 때, 최종 점수를 구하라

[입력]
- 격자 크기 N (3 <= N <= 20)
- 학생 좋아하는 학생번호4개
```

# 2. 설계

```
이 문제는 매번 맵 전체를 탐색하고 빈자리에 대해 인접한 4방향이 좋아하는 친구인지 16번검사해도
충분한 문제다.n이 20일 때 400 * 16 * 400 = 2,560,000

1. 학생 번호를 입력받아 자리를 리턴하는 함수가 있다고 하자
2. 해당 함수를 N*N만큼 반복하는 함수로 학생 위치를 정한다
3. 점수 산정 함수로 최종 결과를 출력한다 -> 하나씩 점수를 산정할게 아니라,
마지막에 한 번에 산정해 줘야함
들어갈때는 좋아하는 학생이 2명이었는데, 마지막에 3명이 될 수도 있음
```
