# 1. 이해

```
[격자판]
- n x m
- 0부터 9까지의 숫자
- 정육면체 하나가 올려져 있음
- 처음 정육면체는 각 면에 0이 쓰여져 있음

[규칙]
- 맵이 0이면 정육면체 바닥 숫자 맵에 복사
- 맵이 0이 아니면 정육면체에 복사 -> 맵 0
- 정육면체는 맵 밖으로 이동 안됨
- 그런 시도가 있으면 무시

[출력]
- 정육면체의 초기 위치와 굴리는 방향이 주어질 때, 이동할 때마다 정육면체의 상단 면에
쓰여져 있는 숫자를 출력하라

[입력]
- n, m, x, y, k ( n : 세로, m : 가로, x, y : 정육면체 위치, k : 굴리기 횟수 )
- 맵
- 굴리기 방향 ( 1 : 동, 2 : 서, 3 : 북, 4 : 남 )
```

# 2. 설계

```
[설계]
- 맵 배열 필요
- 방향 배열 필요
- 주사위를 굴리는 함수 -> 방향을 입력
	- 주사위를 하나의 배열로 매핑할 수 없을까?
	- 했다면 회전도 정의할 수 있음
	- 정육면체를 4x3 매트릭스에 매핑 가능
	- 회전도 동서북남 각각에 대해 정의 가능
- 맵 혹은 주사위를 업데이트하는 함수 -> 굴린 이후 맵 업데이트 필요
```
