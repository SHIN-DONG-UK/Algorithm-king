[문제 요약]

- N x N 격자가 있고, 각 격자는 (1) 먼지, (2) 먼지x, (3) 물건
- 먼지가 있는 공간은 1~100 사이에 먼지 양(p)를 가짐
- 각 로봇 청소기는 초기 위치를 가지며, 해당 위치는 먼지가 없음이 보장됨
- 테스트는 다음과 같은 방법으로 진행됨

1. 청소기 이동
- 각각의 로봇 청소기는 순서대로 이동 거리가 가장 가까운 오염된 격자로 이동
- 물건이 위치한 격자나 청소기가 있는 격자로는 지나갈 수 없음
- 가장 가까운 격자가 여러 개일 경우 행이 작은순, 그래도 여러개면 열이 작은순

2. 청소
- 청소기가 바라보고 있는 방향을 기준으로 본인, 왼쪽, 위쪽, 오른쪽 청소 가능
- 청소할 수 있는 4가지 격자 중 먼지량이 가장 큰 방향에서 청소를 시작함
- 격자마다 청소할 수 있는 최대 먼지량은 20임
- 합이 같은 방향이 여러 개인 경우, 오른쪽->아래쪽->왼쪽->위쪽 순서대로 우선순위 선택
- 청소는 청소기마다 순서대로 진행

3. 먼지 축적
- 먼지가 있는 모든 격자에 동시에 5씩 추가됨

4. 먼지 확산
- 깨끗한 격자에 주변 4방향 격자의 먼지량 합을 10으로 나눈 값만큼 먼지가 확산됨
- 편의상 나눗셈 과정에 생기는 소숫점 아래 수는 버림
- 모든 깨끗한 격자에 대해 동시에 확산이 이루어짐

5. 출력
- 전체 공간의 총 먼지량을 출력
- 먼지가 있는 곳이 없으면 0을 출력


- 위 테스트 과정을 L번 반복함
- 각 테스트가 끝날 떄마다 총 먼지량을 출력하는 프로그램을 작성하라

[제약 사항]

- 2 <= N <= 30
- 1 <= K <= 50
- 1 <= L <= 50
- -1 <= p <= 100 (-1인 경우 물건이 위치한 격자)

[나이브한 생각]

- 모든 격자를 순회하는데 최악의 경우 900
- 1번 -> 900
- 2번 -> 50 * 4 * 4 *  = 800
- 3번 -> 900
- 4번 -> 900
- 5번 -> 1
- 테스트 한 번 = 900 *3 + 800 = 3500
- 3500 * L(=50) = 165,000
- 그냥 빡구현해도 시간 복잡도 충분

[설계 - 알고리즘]

1. 청소기 이동
- 가장 가까운 먼지가 있는 곳으로 이동하는데, 플러드필을 이용한다
- 2차원 격자에서 4방향 탐색하면서 플러드필하다가, 먼지를 만나는 순간 종료하면 된다
- 또, 우선순위가 있으므로 상, 좌, 우, 하 순서대로 탐색한다
- 상좌하우 순서대로 플러드필해서 처음으로 먼지인 곳이 바로 가장 가까운 우선순위가 가장 높은 먼지 위치다

2. 청소
- 우하좌상 순서대로 먼지량을 측정한다
- 최댓값과 그떄의 방향을 기록해둔다
- 그 방향을 기준으로 '청소'를 반영한다

3. 먼지 축적
- 2차원 격자를 돌면서 먼지가 있는 격자에 +5한다

4. 먼지 확산
- 2차원 격자를 모두 돌면서 먼지가 없는 격자를 찾는다
- 주변 4방향을 탐색해서 먼지량을 더한다
- 먼지량 합에 10을 나눈 값을 현재 격자에 기록한다
- 동시에 확산이 이루어져야 하므로, 임시 격자에 먼지량을 기록한다
- 그리고나서 원래 격자에 복사한다

5. 출력
- 2차원 격자를 모두 돌면서 먼지량을 더한다
- 그리고 출력한다

[설계 - 자료구조]

- 로봇 구조체
	- 위치
	- 방향

- 로봇 구조체 벡터 -> 모든 로봇을 관리하는 동적 배열
- 2차원 격자
- 2차원 방문 배열
