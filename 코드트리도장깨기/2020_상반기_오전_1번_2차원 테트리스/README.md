# 1. 이해

```
[문제 요약]

- 특이한 모양의 테트리스 판이 있음
- 3개의 타일을 이용해 게임이 진행됨
- 파란색 보드에서의 위치와 타일의 종류가 주어질 때, 해당 블록은 빨간색 블록과 노란색 블록으로 각각 이동
- 행이나 열이 타일로 가득찬 경우와 연한칸에 타일이 있는 경우가 동시에 발생할 수 있음
    - 이때 점수 획득을 모두 진행한 이후, 연한 칸의 블록을 처리
- 타일의 종류와 놓은 위치가 순서대로 주어질 때 얻은 점수와 노란색 보드와 빨간색 보드에 타일이 있는 칸의 합을 알려주는 프로그램을 작성하라

[제약 사항]

- 블록을 입력한 횟수 k <= 10,000
- 파란색 범위를 벗어나는 블록의 위치는 주어지지 않는다
```

# 2. 설계

```
[설계]

- 격자모양을 어떻게 결정할 것인가?
  - 그냥 2차원 배열을 사용하자 -> 10 x 10
    - 파란색 보드, 빨간색 보드, 노란색 보드는 하드코딩하게 결정됨
    - 연한 구역도 하드코딩으로 결정할 수 있음
    - 파란색 보드 -> (0,0)부터 4 x 4
  - 빨간색 보드 -> (0,6)부터 4 x 4
  - 노란색 보드 -> (6,0)부터 4 x 4
    - 빨간색 연한 칸 -> (0,4)부터 4 x 2
    - 노란색 연한 칸 -> (4,0)부터 2 x 4

- 행 or 열이 가득 찬 것은 어떻게 알 것인가?
    - 열 -> 개수
    - 행 -> 개수
    - 이렇게 개수를 달아놓으면 되지 않나?
    - 문제의 크기가 작아서 매번 검사해도 되긴 할 듯

- 블록을 어떻게 이동시킬 것인가?
    - 각 블록마다 이동 함수를 만든다
    - 반복문으로 검사해서 적당한 위치를 찾고 이동한다

- 가득 차면 어떻게 처리할 것인가?
    - 배열을 땡긴다
```
