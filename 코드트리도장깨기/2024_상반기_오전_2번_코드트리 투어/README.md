[문제 요약]

- 코드트리 랜드는 n개의 도시와 각 도시를 연결하는 m개의 간선으로 이루어짐
- 각 도시는 0번부터 n-1번까지 번호가 붙어져 있고, 간선은 무방향
- 두 도시 사이를 연결하는 간선은 여러 개 존재할 수 있음
- 자기자신을 향하는 간선도 가능
- 출발지는 하나로 통일
- 처음 코드트리 여행 상품의 출발지는 0번 도시
- 다음 5가지 명령에 따라 투어가 진행됨

```
1. 코드트리 랜드 건설

- 코드트리 랜드 관련 정보가 주어짐
- 도시의 수 n, 간선의 수 m, 그리고 간선에 해당하는 정보 (v, u, w)가 주어짐
- (v, u, w)는 도시 v와 도시 u는 가중치 w인 간선으로 연결되어 있음을 의미
- 100 n m v1 u1 w1 v2 u2 w2 .. vm um wm 형태로 입력
- 이 명령은 항상 첫 번째 명령으로만 주어지며, 항상 주어짐


2. 여행 상품 생성

- 코드트리 여행사는 (id, revenue, dest)에 해당하는 여행 상품을 추가로 만들고 이를 관리 목록에 추가
- 여행 상품은 고유한 식별자 id를 가지며, 이 여행 상품을 통해 얻게되는 매출은 revenue, 도착지는 dest
- 200 id revenue dest 형태로 주어짐
- 이를 관리 목록에 추가
- 주어지는 id는 모두 다름


3. 여행 상품 취소

- 고유 식별자 id에 해당하는 여행 상품이 존재하는 경우, 해당 id의 여행 상품을 관리 목록에서 삭제
- 300 id 형태로 주어짐


4. 최적의 여행 상품 판매

- 관리 목록에서 조건에 맞는 최적의 여행 상품을 선택하여 판매함
- 선택 조건은 여행사가 이 상품을 판매함으로써 얻게 되는 이득 [revenue - cost]가 최대인 상품을 우선적으로 고려
- 같은 값을 가지는 상품이 여러 개 있을 경우 id가 가장 작은 상품을 선택
- cost는 현재 여행 상품의 출발지로부터 id 상품의 도착지 dest까지 도달하기 위한 최단거리를 의미
- 만약 출발지로부터 dest에 도달하는 것이 불가능하거나, cost가 revenue값보다 더 커서 여행사가 이득을 얻을 수 없는 상황이라면 이러한 상품들은 판매 불가 상품
- 코드트리는 판매 가능한 상품 중 가장 우선순위가 높은 상품을 1개 판매
- 이 상품의 id를 출력한 뒤 이 상품을 관리 목록에서 제거
- 만약 판매 가능한 상품이 전혀 없다면 -1을 출력하고 상품을 제거하지 않아도 됨
- 400 형태로 주어짐


5. 여행 상품의 출발지 변경

- 여행 상품의 출발지를 전부 s로 변경하는 명령
- 출발지가 변경됨에 따라 각 상품의 cost가 변경될 수 있음
- 500 s 형태로 주어짐

```

- Q번에 걸쳐 명령을 순서대로 진행하며 알맞은 답을 출력하라


[제약 사항]

- Q <= 100,000
- n <= 2,000
- m <= 10,000
- w <= 100
- id <= 30000
- revenue <= 1,000
- 생성 <= 30,000
- 취소 <= 30,000
- 최적의 여행 상품 판매 <= 30,000
- 출발지 변경 <= 15


[나이브한 생각]

- 먼저 그래프의 구조는 변하지 않는다
- 노드의 개수는 최대 2,000개이며, 간선은 최대 1만개이다
- 가중치가 있는 그래프에서 최단거리는 다익스트라를 활용해야 함
- 다익스트라는 O(NlogN) 복잡도임
- 최악의 경우, 한 번 다익스트라를 돌리면 2만정도의 반복을 수행함
- 여행 상품이 3만개 있다고 하자
- 최적의 여행 상품 판매를 수행하면, 다익스트라를 한 번 돌리고->3만개의 상품 중 최적의 하나를 찾아야 함
- 한 번 수행하는데 2만 * (3만개 정렬)이 소요됨
- 이를 위해, 여행 상품을 생성할 때부터 미리 정렬된 자료를 갖고 있어야 함
- 그렇게 하면 단순히 정렬된 자료구조에서 하나 뽑으면 됨
- 단, 출발지가 변경될 경우 미리 만들어 둔 정렬된 자료는 쓰레기값이 됨
- 출발지의 변경 횟수는 15회로 매우 적으므로, 정렬된 자료구조를 유지하다가 출발지가 변경되면 새로 다익스트라를 돌리고 정렬된 자료구조를 새로 만들면 된다
- 정렬된 자료구조는 한 번 만드는데 NlogN이 소요된다
- 최악의 경우라도 15 * 30000 log(30000)이다
- 출발지가 변경되기 전까진 다익스트라를 한 번 돌려서 정렬된 자료구조를 갖고 있으면 된다
- 삭제되었는데 정렬된 자료구조에는 반영되지 않을 수 있다
- 이러한 lazy deletion을 반영하기 위해서는 원본 배열을 갖고 있고, 이 원본 배열에서 isRemoved를 관리하고 있어야 한다
- 그래서 정렬된 자료구조에서 뽑았을 때 최신 반영이 가능하다

[설계]

<자료구조>

- 간선 구조체 Edge
	- int n
	- int dist

- 다익스트라용 구조체 Item
	- int n
	- int dist

- 여행 상품 구조체 Product
	- int revenue
	- int dest
	- bool alive

- 정렬된 자료구조용 구조체 PQNode
	- int profit
	- int id

- 그래프 인접리스트 방식
	- 각 노드는 0번부터 n-1번까지 번호가 붙어 있으므로 vector<Edge> adjList[MAX_N]으로 선언

- best 배열
	- 각 노드 번호에 해당하는 1차원 배열
	- 다익스트라 시작 전, 모든 값을 무한대(1e9)로 초기화해야 함

- 정렬된 자료구조
	- 우선순위1: [revenue - cost]가 큰 순
	- 우선순위2: id가 작은 순
	- Priority Queue로 만들자

- 관리 목록 배열
	- id는 3만 이하이고, 모든 id가 서로 다르므로 단순 배열을 사용해서 관리 목록을 관리해도 된다
	- hash map보다 배열이 시간적으로 더 이득이다
	- 처음에 모든 id는 alive = false

4. 최적의 여행 상품 판매 (400)

- 만약 PQ의 top이 삭제되었으면(alive = false), 다음 애를 찾아보자
- 만약 PQ의 top이 1e9이면 도달할 수 없는 상품
	- 1e9가 아닐 때까지 다음 상품을 찾아야 함
- 만약 PQ의 top이 음수이면 나머지 애들은 모두 판매 불가 상품
	- 더 이상 찾지 않음
- 만약 PQ가 비어있으면, 판매 가능한 상품이 존재하지 않음
	- 더 이상 찾지 않음
- PQ에서 온전히 top을 뽑았으면, 판매 가능한 상품
	- 이 상품의 id로 접근해 alive = false
	- 이 상품의 id를 출력
- PQ에서 온전히 top을 뽑지 못했으면, -1을 출력
