- 마을 시장 코디는 다음 명령을 내릴 수 있음

```
1. 마을 상태 확인

- N: 거리의 크기
- M: 초기에 존재하는 가로등의 개수
- L1, ..., L_M: 초기에 존재하는 각 가로등의 위치 정보
- 가로등의 좌표는 오름차순으로 주어지며, 주어진 순서대로 1, 2, ..., M번의 고유번호가 부여됨

2. 가로등 추가

- 초기에 설치되어 있는 M개의 가로등 이후에 추가로 설치하는 명령
- 인접한 가로등 사이의 거리가 가장 먼 곳의 가운데에 새로운 가로등을 설치
	- 인접한 가로등의 거리가 가장 먼 곳이 여러개 존재한다면, 좌표 값이 작은 가로등 쌍을 선택
- 새로 설치할 가로등은 반드시 서로 다른 두 가로등 사이에 위치

3. 가로등 제거

- D번 가로등을 제거
- 가로등이 2개 이하일 경우 해당 명령은 주어지지 않음
```

[제약 조건]

- 쿼리개수 Q <= 10만
- 마을 거리 N <= 10억
- 초기 가로등 개수 M <= 10만

[나이브한 생각]

- 마을의 거리가 최대 10억이므로, 단순 배열을 선언해서 문제를 해결할 수는 없다
- 인접한 가로등 사이 거리를 기준으로 새 가로등을 추가해야 함
- heap을 활용하면 O(logN)만에 자료를 새로 추가할 수 있다
- 그리고 정렬을 유지할 수 있다
- 두 가로등 사이 거리가 작은 순서대로 자료구조에 저장하자

-> 가로등 추가 명령
- heap에서 머리를 하나 따자 == 가로등 사이 거리가 가장 작은 값
- 머리에는 왼쪽 가로등 포인터와 오른쪽 가로등 포인터가 들어있음
- 이 둘의 위치를 갖고 새로 설치할 가로등의 위치 계산
- 그리고 두 개의 item을 새로 추가
	- 하나는 왼쪽 가로등과 새거
	- 다른 하나는 새거와 오른쪽 가로등

-> 가로등 제거 명령

- heap에서 찾아서 삭제하는 건 무리
- 논리적 삭제 이후 추가 명령에서 dummy를 삭제하는 식으로 설계해보자
- 논리적 삭제는 단순히 해당 가로등을 isRemoved = true 처리하는 것

-> 최소 전력 계산

- r이 가능한 구간을 탐색해서 절반씩 줄여나가는 방식으로 찾아보자
- r값은 최대 10억이다
- 그리고 r값을 줄여도 현재 가로등으로 충분한지 1차 loop를 순회하여 체크할 수 있다
- r을 줄였는데 가능하면, 절반으로 줄이자
- 그래도 가능하면, 또 절반으로 줄인다
- 근데 불가능하면, 이번에는 오른쪽 구간을 탐색해야한다
- 이런 Prametric Search를 활용하면 문제를 해결할 수 있을 것 같다

- bool check
	- 두 가로등 사이 거리가 가장 먼 것만 확인해봐도 2r이 가능한지 아닌지 확인할 수 있다
