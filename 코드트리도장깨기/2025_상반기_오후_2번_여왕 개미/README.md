[문제 요약]

- 1차원 수직선으로 표현되는 땅에 여왕 개미와 일 개미들이 개미집을 집고자 한다
- 좌표는 0이상 10억 이하의 정수로 표현
- 여왕 개미는 일 개미에게 다음과 같은 명령을 함

```
1. 마을 건설 (100)

- 입력 형식: 100 N x1 x2 x3 ... xN
- 여왕 개미의 집을 x=0 위치에 건설
- N개의 개미집을 건설하는데, 1~N까지 번호로 표현되고 i번 개미의 위치는 xi
	- 마을의 개미집은 번호가 큰 것이 더 큰 x좌표를 가짐 즉, 오름차순으로 주어짐
- 마을 건설은 초기에 단 한 번만 주어짐

2. 개미집 건설 (200)

- 입력 형식: 200 p
- 새로운 개미집을 하나 건설
- 건설할 개미집의 위치는 양의 정수 p로 주어짐
- p는 이제까지 건설된 모든 개미집의 좌표보다 큰 값으로 주어짐
	- 개미집 건설 명령 중 k번째 명령으로 주어지는 개미집의 번호는 N + k번으로 표현됨


3. 개미집 철거

- 입력 형식: 300 q
- 개미집을 하나 철거
- 철거할 개미집의 번호 q가 주어짐
- q번 개미집이 이미 철거된 상태거나, 아직 지어지지 않은 경우는 입력으로 주어지지 않음

4. 개미집 정찰

- 입력 형식: 400 r
- 일 개미가 정찰을 진행해 모든 개미집을 확인함
- 정찰을 나갈 개미의 수 r이 주어짐
- r마리의 개미가 서로 다른 개미집을 선택
	- 여왕 개미가 사는 집을 선택할 수 있음
- 선택을 마치면 각 개미가 x값이 증가하는 방향으로 1초에 1만큼 계속 이동
	- 처음 선택 위치를 포함해서, 개미가 지나가는 모든 개미집은 '안전한 개미집'
	- 단, 여왕 개미집은 정찰하지 않아도 항상 '안전한 개미집'
- 각 개미는 자신이 x값이 증가하는 방향으로 더 진행해도 갈 수 있는 개미집이 없거나, 처음 만나는 개미집이 '안전한 개미집'인 경우에는 이동을 중지함
	- 따라서 모든 개미집이 '안전한 개미집'이 되면 모든 이동은 종료됨
- 일 개미들은 처음 개미집을 선택할 때, 정찰에 걸리는 시간이 최소가 되도록 개미집을 적절히 선택함
```

- 초기 개미집의 위치와 여왕 개미의 명령이 주어진다
- 개미집 정찰 명령이 주어질 때마다 여왕 개미가 보고 받은 시간을 출력하라

[제약 사항]

- 여왕 개미 명령 총 개수 Q: 2 <= Q <= 20,000
- 초기 개미집 개수 N: 1 <= N <= 10,000
- 개미집 건설 명령 <= 10,000
- 개미집 철거 명령 <= 10,000
- 개미집 정찰 명령 <= 100


[나이브한 생각]

- 개미집 정찰 명령이 시간이 많이 소요되는 함수이므로, 이 함수를 기준으로 설계해보자
- r마리의 개미가 모든 개미집을 정찰해야 한다
- r마리를 적절히 개미집에 분배하여 걸리는 시간이 최소가 되어야 한다
- 걸린 시간을 계산할 때, x를 1씩 증가할 필요는 없고, 개미집을 1씩 증가시키는 방식이 필요하다
- 개미가 이동할 수 있는 시간(거리)의 상한을 제한하면 되지 않나?
- 그 상한을 제한해놓고 개미를 이동시켜보는거다
- 그랬을 때 가능하면 상한을 줄이고, 불가능하면 상한을 늘리는 방식으로 찾아나가면 되겠다
- 이렇게 하면 이동시키는 시간복잡도 * log10억이다
	- 거의 이동시키는 시간복잡도만 고려하면 된다
	- 이동시키는 시간복잡도도 최대 2만
- 결국 Parametric Search를 수행하면 되겠다
