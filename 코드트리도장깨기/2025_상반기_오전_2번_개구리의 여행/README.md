[문제 요약]

- 호수에 개구리가 한 마리 살고 있다
- 호수는 N x N 크기의 정사각형
- 호수에 돌이 있고 개구리는 돌 위에서 돌아다닐 수 있다
- 돌의 위치는 (1,1) ~ (N, N)
- 돌의 종류
	- 안전한 돌 == .
	- 미끄러운 돌 == S
	- 천적이 사는 돌 == #
- 개구리의 여행은 시작 위치와 도착 위치를 정한 뒤, 시작 위치에서 도착 위치로 가는 것
- 개구리의 초기 점프력은 1이며, 여행 과정에서 다음과 같은 행동을 할 수 있음
```
1. 점프

- 현재 위치에서 상하좌우 방향 중 하나를 골라 '현재 점프력'만큼 칸을 이동
- 단, 이동하려는 위치에 돌이 없다면 그 위치로는 이동이 불가능
- 도착한 위치에 '미끄러운 돌'이 있어도 불가능
- 점프해서 도착하는 위치나 점프하면서 지나치는 경로에 '천적이 사는 돌'이 있는 경우도 불가능
- 점프에는 1만큼의 시간이 소요

2. 점프력 증가

- 점프력을 1만큼 증가시킴
- 기존 점프력이 1,2,3,4 중 하나인 경우에만 가능
- 즉, 최대 점프력은 5
- 점프력 증가 후 점프력을 k라고 할 때, k^2만큼 시간이 소요

3. 점프력 감소

- 점프력을 감소시킴
- 기존 점프력이 k인 경우 1, 2, ..., k-1 중 하나의 점프력을 갖도록 만들 수 있음
- 점프력 감소에는 1만큼의 시간이 소요
```
- 개구리는 이 호수에서 총 Q번의 여행 계획을 세움
- 개구리는 각 계획마다 최대한 짧은 시간에 여행을 마치고자 함
- 호수에 있는 돌의 정보와 개구리의 여행 계회들이 주어질 때, 각 여행에 걸리는 최소 시간을 출력하라
- 만약 도착이 불가능한 계획이 있다면 그 계획에 대한 답으로 -1을 출력

[제약 사항]

- N <= 50
- Q <= 1,000
- 출발지와 도착지는 항상 안전한 돌

[나이브한 생각]

- 같은 칸에는 여러 경우로 도달할 수 있다
- 예를 들어, 점프력 1로 도달할 수도 있고, 점프력 3으로 도달할 수도 있다
- 같은 칸에 여러 상태로 도달할 때, 이중 최적을 찾는 방식을 적용해야 한다
- 매번 최적의 선택을 하여 최종 목적지에 도달하기 위해, 다익스트라 포맷을 활용해보자
- 이때 '최단거리' == '걸린 시간'이다
- 그럼 현재 '상태'에서 '가능한 모든 상태'로 전이해봐야 한다
- 그리고 그 중 '최단 거리'를 그리디하게 뽑아서 이동하는 방식으로 설계해보자
- 그리고 '더미' 상태 처리를 잘 해줘야 한다

- 2차원 보드
	- MAX_N 50

	- 출발점과 도착점 노드를 입력받는다
	- 출발점 위치와 비용을 우선순위 큐에 집어 넣는다
	- 우선순위 큐에서 top 노드를 뽑는다
		- best 테이블보다 비용이 크면, 그건 '더미'다
	- top 노드에서 가능한 모든 상태(다음 노드)를 우선순위 큐에 push한다
		- 점프력을 1~5로 만들고, 각 점프력마다 상하좌우 방향으로 '점프'하는 상태들
