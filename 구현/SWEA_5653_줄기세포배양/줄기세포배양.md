# 1. 이해

```
[줄기세포]
- 초기 상태에서 줄기 세포들은 비활성 상태
- 생명력 수치가 X인 줄기 세포의 경우 X시간 동안 비활성 상태이고 X시간이 지나는 순간 활성 상태
- 활성 상태가 되면 X시간 동안 살아있을 수 있으며 X시간 이후 죽게됨

[활성 상태]
- 활성화된 줄기 세포는 첫 1시간 동안 상,하,좌,우 네 방향으로 동시에 번식
- 번식된 줄기 세포는 '비활성 상태'
- 이미 줄기 세포가 존재하면 번식 x
- 두 개 이상의 줄기 세포가 하나의 그리드 셀에 동시 번식하려고 하는 경우,
생명력 수치가 높은 줄기 세포가 해당 그리드 셀을 혼자서 차지

[출력]
- 줄기 세포의 초기 상태 정보와 배양 시간 K시간이 주어질 때,
K시간 후 살아있는 줄기 세포(비활성 상태 + 활성 상태)의 총 개수를 구하라

[입력]
- Test Case T
- N (max50) M (max50) K(max300)
- 초기 맵상태
```

# 2. 설계

```
[알고리즘]
- 주어진 대로 구현하면 되는 시뮬레이션이다
- 그런데 배양 용기의 크기가 무한하다는 설정이 매우 생소하다
- 번식을 수행하기 위해서는 이 배양 용기가 필요한데, 따로 주어지지 않는다
- 하지만 K가 최대 300이므로, 50+300을 넘을 수 없다
- 따라서 배양 용기의 크기를 500으로 설정하고 시작해도 무방하다
- 시간 초과 방지를 위해 왼쪽 위 모서리 좌표를 매번 업데이트해서 여기부터 탐색할 필요가 있다
- 맵의 가운데 초기 상태 정보를 저장하고 시뮬레이션하자

[시뮬레이션]
- 용기의 한 가운데 초기 상태가 저장되어 있다
	- 맵 최대 사이즈 501
	- 맵 중앙 좌표 = (250, 250)
	- 초기 상태 왼쪽 위 모서리 좌표 = (250 - N/2, 250 - M/2)
	- 초기 상태 오른쪽 아래 모서리 좌표 = (250 + N/2, 250 + M/2)
		- 단, N,M이 홀수일때와 짝수일 때 다름
		- N, M이 짝수면 -1해줘야 함
- 2중 for문
	- 활성상태에 있는 줄기 세포를 번식한다
	- 비활성 상태의 줄기 세포를 업데이트한다
	- 왼쪽 위 모서리 좌표를 업데이트 한다
```
