# 1. 이해

```
[핀볼 게임]
- N x N 맵
- 정사각형 블록과 4가지 형태의 삼각형 블록
- 웜홀과 블랙홀

[핀볼]
- 작은 핀볼 하나가 상하좌우 중 한 방향으로 움직임
- 핀볼은 블록이나 웜홀 또는 블랙홀을 만나지 않는 한 현재 방향을 유지하면서 계속 전진
- 수평면이나 수직면을 만나면 반사, 경사면을 만나면 직각 방향으로 경로 변경
- 벽을 만나도 반사
- 웜홀에 빠지면 동일한 숫자를 가진 다른 반대편 웜홀로 빠져나오고 진행방향 유지
	- 웜홀은 반드시 쌍으로 주어짐
	- 6~10 사이의 숫자
- 핀볼이 블랙홀을 만나면, 핀볼이 사라지게 되어 게임이 끝남

[게임 종료]
- 핀볼이 출발 위치로 돌아오거나, 블랙홀에 빠질 때

[점수]
- 벽이나 블록에 부딪힌 횟수

[시작 위치]
- 출발 위치와 방향을 임의로 선정 가능

[출력]
- 게임에서 얻을 수 있는 점수의 최댓값을 구하라
- 단, 빈공간에서 출발 가능

[입력]
- N (max 100)
- 맵정보
	-1 : 블랙홀
	0 : 빈공간
	1~5 : 블록
	6~10 : 웜홀
```

# 2. 설계

```
[알고리즘]
- 주어진 대로 구현하면 되는 시뮬레이션 문제이다
- 단, 점수의 최댓값을 구하라고 하는데, 언제 최대가 될 지 알 수 없으므로 모든 경우를 다 해보는
Brute Force 알고리즘을 구현한다

[설계]
- 출발 위치와 방향을 입력하면 시뮬레이션하는 함수가 있다고 가정한다
- 2중 for문을 돌면서 빈공간에서 4방향에 대해 시뮬레이션 함수를 돌린다
- 시뮬레이션 함수를 모두 돌려서 나오는 가장 큰 점수를 출력한다

[시뮬레이션 함수]
- 어떤 번호의 블럭에 어떤 방향으로 진입하면 방향을 반환하도록 하는 2차원 배열을 선언하자
- 이렇게 하면 다음 방향을 얻기 쉽다
- 다음 위치를 현재 위치와 방향으로 계산한다
	1. 다음 위치가 빈 공간이면 그냥 직진한다
	2. 다음 위치가 블록이면 전진하고 방향을 바꾼다
		- 여기에 맵 경계를 5로 감싸서 맵 밖으로 이동하는 것을 블록으로 퉁쳐서 고려하였다
	3. 다음 위치가 웜홀이면 반대편 웜홀로 이동한다
	4. 다음 위치가 블랙홀이면 리턴한다
	5. 다음 위치가 시작위치면 리턴한다
	
```
