# 1. 이해

```
[게임 규칙]
- 특별한 모양의 보드가 있음
- 이 보드는 빨간색, 초록색, 파란색 영역이 있음
- 빨간색 영역에 3가지 모양의 블록을 놓을거임
	- 1x1, 1x2, 2x1 세 가지가 있음
- 빨간색에 블록을 놓으면 초록색 영역과 파란색 영역으로 경계 혹은 다른 블록을 만날 때까지 이동함
- 이동 다하고 초록색 영역에서 하나의 행이 블록으로 가득차면, 
그 행 사라지고 위에 있는 블록은 한 칸 내려오고 1점을 획득함
- 이동 다하고 파란색 영역에서 하나의 열이 블록으로 가득차면,
그 열 사라지고 왼쪽에 있는 블록은 한 칸 움직이고 1점 획득함
- 초록색 영역과 파란색 영역은 특별한 행과 열이 있음
- 만약 블록이 이 특별한 행과 열에 존재하면 존재하는 행 또는 열만큼 블록이 한 칸씩 이동함
- 행이나 열이 타일로 가득찬 경우와 특별한 칸에 블록이 있는 경우가 동시에 발생할 수 있음
- 이 경우 행이나 열이 가득 찬 경우가 없을 떄까지 점수를 획득하고, 그 다음에 특별한 칸을 처리해야 함

[출력]
- 블록을 놓은 위치가 순서대로 주어졌을 떄, 얻은 점수와 초록색 보드와 파란색 보드에 타일이
있느 개수를 모두 출력하라

[입력]
- 블록을 놓은 횟수 N
- N개의 줄에 블록을 놓은 정보
	- t x y
	- t : 블록 번호
	- (x, y) : 머리 위치
```

# 2. 설계

```
사실 이 문제는 그대로 구현하면 되는 시뮬레이션 문제이다
따라서 시간복잡도를 고려하지 않고 주어진 조건을 잘 만족하면 된다

- 먼저 필요한 함수를 생각해보자
- 블록 타입과 기준점 위치가 주어지면 블록을 초록색 영역과 파란색 영역으로 이동시키는 함수가 필요
- 점수를 체크하는 함수가 필요
- 특별한 영역의 이동을 처리하는 함수가 필요
- 전체 과정을 반복할 함수가 필요
```
