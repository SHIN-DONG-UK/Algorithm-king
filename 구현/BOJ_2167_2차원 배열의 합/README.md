## 💡 BOJ 2167 - 2차원 배열의 합[[문제 링크](https://www.acmicpc.net/problem/2167)]

| 항목 | 내용 |
|------|------|
| 문제 유형 | `구현` |
| 사용 알고리즘 | `누적 합` |
| 난이도 | `실버5` |

### ✅ 문제 설명
**[문제 요약]**

- 2차원 배열이 주어졌을 때, (i, j) 위치부터 (x, y) 위치까지에 저장되어 있는 수들의 합을 구하라
- (i, j) 위치 == i행 j열

**[제약 사항]**

- 1 <= N, M <= 300
- 주어지는 수의 절댓값은 10,000이하
- 합을 구할 부분의 개수 K <= 10,000
- 배열의 합은 2^31-1보다 작거나 같다

### 🔍 풀이 접근

- 시간복잡도를 계산해보자
- 최악의 경우 300*300*10,000 = 900,000,000
- 따라서 한 번 계산하는데 O(NM)은 안됨
- 사실 O(1)만에 구할 수 있음
- 누적합 배열을 만들어놓으면 됨

### 💡 핵심 로직

- S[i][j] = (1,1)부터 (i,j)까지의 합이라고 정의하자
- 그럼 우리가 구하는 (i,j) 위치부터 (x,y) 위치까지의 합 T는 S로 표현하면 다음과 같다

T = S[x][y] - S[i-1][y] - S[x][j-1] + S[i-1][j-1]

(예)
0 0 0 0
0 1 2 3
0 4 5 6
0 7 8 9

(S)
0 0 0 0
0 1 3 6
0 5 12 21
0 12 27 45
