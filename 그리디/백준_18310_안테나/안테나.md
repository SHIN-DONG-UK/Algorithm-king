# 1. 이해

```
[상황]
- 일직선 상에 여러 채의 집
- 특정 위치에 안테나 놓고 싶음
- 안테나로부터 모든 집까지의 거리 합이 최소가 되고자 함
- 동일한 위치에 여러 집 가능
- 안테나는 집이 있는 위치에만 설치 가능

[출력]
- 안테나 위치 출력
- 여러 개인 경우 가장 작은 값 출력

[입력]
- 집의 수 N (1≤N≤200000)
- 집 위치
```

# 2. 설계

```
- 이거, N이 십만이라서 2중 for문 안됨
- 1중만에 가능하다는 소리
- 꼭 2중 for문으로 검사할 필요가 있을까?
- 수식으로 가능함

[의사 코드]
1. 오름차순 정렬
2. 첫 번째 집에 안테나 두고 계산
3. 오른쪽으로 옮기면서 값 업데이트
4. 1중 for문 돌면서 업데이트된 값의 최소를 출력

[예제 검증]
4
5 1 7 9
1 5 7 9
antena = 0
sum = 0+4+6+8=18

antena = 1
sum = 4+2+4=10

antean = 2
sum = 6+2+2=10

antena = 3
sum = 8+4+2=14

정담 : 10
```

# 3. 설계 오류

```
- 집이 중복이 있는 경우를 고려하지 않음
- 이 문제는 그냥 '중간값'을 구하면 됨
```
