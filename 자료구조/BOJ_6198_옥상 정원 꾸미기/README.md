## 💡 BOJ 6198 - 옥상 정원 꾸미기 [[문제 링크](https://www.acmicpc.net/problem/6198)]

| 항목 | 내용 |
|------|------|
| 문제 유형 | `자료구조` |
| 사용 알고리즘 | `Monotonic Stack` |
| 난이도 | `골드5` |

### ✅ 문제 설명
**[문제 요약]**

- 도시에 N개의 빌딩이 있다
- i번째 빌딩의 키가 hi이고, 모든 빌딩은 일렬로 서 있고 오른쪽으로만 볼 수 있다
- 자신이 위치한 빌딩보다 높거나 같은 빌딩이 있으면 그 다음에 있는 모든 빌딩의 옥상은 보지 못한다
- 각 관리인들이 벤치마킹이 가능한 빌딩의 수의 합을 출력하라

**[제약 사항]**

- N <= 80,000
- 각 빌딩의 높이 <= 10억

### 🔍 풀이 접근

- O(N^2)의 시간복잡도로 문제를 해결할 수 없다
- 단조스택을 활용해 나를 볼 수 있는 건물들만 스택에 남기자
- 즉, 누적을 활용해 효율적으로 빌딩을 관리한다

### 💡 핵심 로직
- 스택에 나를 볼 수 있는 건물들만 스택에 남긴다
- 즉, 현재 빌딩의 높이가 스택의 top보다 크거나 같으면 나를 볼 수 없으므로 pop한다
- 현재 빌딩의 높이가 스택의 top보다 작으면 나를 볼 수 있으므로 스택에 유지한다
- 스택에 남아있는 빌딩의 개수는 나를 볼 수 있는 빌딩의 개수다
- 이후 현재 빌딩 높이를 스택에 push한다

### 📌 배운 점
- 단조 스택을 유지하여 나를 볼 수 있는 빌딩의 수를 유지하는 방법을 이해했습니다.
- 문제에서 제시한 관점은 내가 볼 수 있는 건물의 수를 더하는 것이었는데, 알고리즘에서는 나를 볼 수 있는 건물을 누적하는 관점이 필요했습니다. 이를 통해 아이디어를 떠올리기 위해 문제를 바라보는 관점을 바꿀 필요가 있음을 공부했습니다.
