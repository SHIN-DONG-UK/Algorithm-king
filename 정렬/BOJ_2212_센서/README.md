## 💡 BOJ 2212 - 센서 [[문제 링크](https://www.acmicpc.net/problem/2212)]

| 항목 | 내용 |
|------|------|
| 문제 유형 | `정렬` |
| 사용 알고리즘 | |
| 난이도 | `골드5` |

### ✅ 문제 설명
**[문제 요약]**

- 도로에 N개의 센서가 직선 위에 있다
- 이 도로에는 최대 K개의 집중국을 세울 수 있다
- 집중국은 각 센서의 수신 가능 영역을 조절할 수 있다
- N개의 센서가 적어도 하나의 집중국과 통신 가능해야 한다
- 각 집중국의 수신 가능 영역의 길이 합이 최소가 되어야 한다

**[제약 사항]**

- 모든 센서의 좌표가 다를 필요는 없다
- 집중국의 수신 가능영역의 길이는 0이상이다
- 센서의 개수 N <= 10,000
- 집중국의 개수 K <= 1,000
- |센서의 좌표| <= 1,000,000

### 🔍 풀이 접근

- 군집을 나누는 문제임
- 파라메트릭 서치로 접근하면, 반례가 존재함
- 모든 군집을 똑같은 상한을 갖도록 하는 건 잘못된 접근법
- 센서 위치를 오름차순으로 정렬하자
- a[마지막] - a[처음]에서 군집을 끊을수록 합이 작아진다
- 군집을 끊었을 때, 보상되는 크기는 인접한 위치 차이인 diff이다
- 이 diff값이 큰 순서대로 K-1번의 군집을 끊으면 최적해가 된다

예)
pos_arr -> 1 3 6 7 9
diff -> 2 3 1 2

### 💡 핵심 로직

- pos_arr를 중복을 제거하고 오름차순 정렬한다
- sum = pos_arr[last] - pos_arr[first]
- diff를 구한다
- diff를 내림차순 정렬한다
- diff의 0번 인덱스부터 K-1개를 sum에서 뺀다

### 📌 배운 점
- 반례로부터 접근법이 잘못되었음을 이해하고, 문제를 해결할 수 있는 아이디어를 구현하는 연습을 했습니다.
