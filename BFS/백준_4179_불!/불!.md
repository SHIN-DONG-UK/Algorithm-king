# 1. 이해

```
[조건]
- 지훈이와 불은 매 분마다 한 칸씩 수평 또는 수직으로 이동
- 불은 각 지점에서 네 방향으로 확산됨
- 지훈이는 미로의 가장자리에 접한 공간에서 탈출할 수 있음
- 지훈이와 불은 벽이 있는 공간은 탈출하지 못함

[출력]
- 지훈이가 불이 도달하기 전에 미로를 탈출할 수 없는 경우 IMPOSSIBLE 출력
- 미로를 탈출할 수 없는 경우 가장 빠른 탈출시간을 출력

[입력]
-  R, C
	R : 행 MAX_R 1000
	C : 열 MAX_C 1000
- 미로
	# : 벽
	. : 지나갈 수 있음
	J : 지훈이의 초기 위치
	F : 불이 난 공간
```

# 2. 설계

```
1. 지훈이와 불을 매 턴마다 업데이트 해야 한다
2. 업데이트 되는 맵에서 BFS를 수행하면 될 것 같다
3. 지훈이가 먼저 움직이고, 불을 이동시키여 한다
	왜냐하면, 동시에 움직이는데 불이 번지고 지훈이가 이동하면
	지훈이가 원래는 갈 수 있는데 못가게 되는 경우가 발생하기 때문

[지훈]
- 지훈이 위치를 전역으로 선언한다
- 지훈이 위치가 시작점이다

[불]
- 불 좌표를 갖고 있다가 이 배열을 쭉 돌면서 각 4방향 맵 업데이트
및 좌표 배열도 업데이트 한다

[BFS]
- 큐에서 갈 수 있는 노드를 꺼낼 때마다, 그 전에 맵을 업데이트 한다
```

# 3. 설계 오류

```
위와 같이 로직을 작성하면 "같은 레벨"에서 방문하는 노드가 "같은 맵"에서 동작할 수 없음
그니까 cp에서 4방향 뽑아서 그 중 하나를 간다
근데 불은 버뜨려져 있다
그래서 같은 레벨에서 검사해야 하는 포인트들이 서로 다른 레벨에서 검사된다
그니까 bfs가 아니게 된다
즉, 최단 거리가 아니게 된다

[해결]
1. 불 BFS를 돌려서 불이 도달하는 시간 맵을 미리 얻어 놓는다
2. 이 맵을 보고 갈 수 있는지 없는지 판단한다
3. 같은 시간에 불과 지훈이가 같이 있는 곳은 갈 수 없다
```
