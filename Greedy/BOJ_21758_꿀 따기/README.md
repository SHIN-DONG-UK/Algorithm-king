## 💡 BOJ 21758 - 꿀 따기 [[문제 링크](https://www.acmicpc.net/problem/21758)]

| 항목 | 내용 |
|------|------|
| 문제 유형 | `Greedy` |
| 사용 알고리즘 | |
| 난이도 | `골드5` |

### ✅ 문제 설명

**[문제 요약]**

- N개의 장소가 있고 각 장소에는 꿀의 양이 적혀 있음
- 장소들 중 서로 다른 두 곳을 골라 벌을 한 마리씩 품
- 또 다른 장소에는 벌통을 둠
- 두 마리 벌은 벌통으로 똑바로 날아가면서 지나가는 모든 칸의 꿀을 땀
- 단, 시작지점은 꿀 못 땀
- 이때 딸 수 있는 꿀의 양의 최댓값을 구하라

**[제약 사항]**

- 3 <= N <= 100,000
- 각 장소의 꿀의 양은 1만 이하 자연수

### 🔍 풀이 접근

- 멀면 멀수록 최적이라, 벌통과 벌 한마리는 무조건 양 끝에 있는게 최적일 수 있다
- 혹은 벌 두 마리는 양 쪽 끝에 있고, 벌통 하나가 가운데 어딘가 있는 경우도 최적일 수 있다
- 다음 세 가지 케이스만 탐색하면 된다
   1. 벌-----벌통, 나머지 벌 어딘가 <- O(N)
    2. 벌통-----벌, 나머지 벌 어딘가 <- O(N)
    3. 벌-------벌, 벌통 어딘가 <- O(N)
- 누적합배열을 만들어서 계산해보자

### 💡 핵심 로직

- 누적합 배열을 만든다
- 세 가지 케이스 각각에 대한 최댓값을 구한다
- 전체 최댓값을 갱신한다
- 합을 계산할 때, 벌이 있는 곳은 합에 넣지 않도록 주의한다

### 📌 배운 점
- 모든 경우의 수는 O(N^3)이지만 결국 최댓값은 그리디하게 결정되므로 3가지 케이스만 O(N)번 하면 되는 문제를 이해했습니다.
