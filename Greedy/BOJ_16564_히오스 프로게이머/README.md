## 💡 BOJ 16564 - 히오스 프로게이머 [[문제 링크](https://www.acmicpc.net/problem/16564)]

| 항목 | 내용 |
|------|------|
| 문제 유형 | `Greedy` |
| 사용 알고리즘 | `정렬` |
| 난이도 | `실버1` |

### ✅ 문제 설명
**[문제 요약]**

- 총 N개의 캐릭터가 있음
- 각 캐릭터의 레벨은 Xi임
- 레벨을 총 합 K만큼 올릴 수 있음
- 팀 목표레벨 T = min(Xi)임
- 최대 팀 목표 레벨을 구하라

**[제약 사항]**

- 1 <= N <= 1,000,000
- 1 <= K <= 1,000,000,000

### 🔍 풀이 접근
- 그냥 우선순위 큐를 사용하면 되지 않나?
- 우선순위 큐에 집어넣고 가장 작은 값을 뽑아서 그걸 더한 다음 다시 우선순위 큐에 집어넣는식으로 해서 항상 최솟값을 뽑아서 거기에 더하는 방식을 취하면 최소 레벨의 최댓값이 나올 것

### 💡 핵심 로직
**[최적화]**

- K가 10억이므로, 10억번 x log N 연산이 수행된다
- 따라서 시간초과 발생
- 계속 1씩 늘려야 할까?
- 최댓값보다 작은 애들이 모두 최댓값과 같아지기 위한 레벨의 총합이 K보다 큰지 작은지를 따지면 되지 않을까?
- 즉, 정렬하고 그리디하게 접근하는 방식을 생각해볼 수 있다
- 만약 i번째 까지 커버 가능하면 i+1번째까지 커버 되는지 해본다
- 그렇지 않으면, 남은 K를 i로 나눈 몫을 구한다
   - i번째값 + 몫이 정답
- i == N-1가 되면, K에서 필요한 레벨 총합을 빼고 남은 만큼 N으로 나눠서 몫을 구한다
   - 최댓값 + 몫이 정답

### 📌 배운 점
- 나이브한 생각으로부터 시간 복잡도를 줄이기 위해 정렬과 그리디를 활용했습니다
