## 💡 BOJ 9009 - 피보나치 [[문제 링크](https://www.acmicpc.net/problem/9009)]

| 항목 | 내용 |
|------|------|
| 문제 유형 | `Greedy` |
| 사용 알고리즘 |  |
| 난이도 | `실버1` |

### ✅ 문제 설명
**[문제 요약]**

- 양의 정수는 하나 혹은 그 이상의 서로 다른 피보나치 수들의 합으로 나타낼 수 있다
- 하나의 양의 정수를 최소 개수의 서로 다른 피보나치 수들의 합으로 나타낼 것

**[제약 사항]**

- 정수 n <= 10억
- 피보나치 수들을 증가하는 순으로 출력할 것

### 🔍 풀이 접근
- 일단 피보나치 수를 미리 만들어 놔야 한다
- 이게 몇 개인지 세어보자
   - n제한에 따라 피보나치 수 40개면 충분
- 따라서 팩토리얼 수준 혹은 2^n수준은 TLE
- 그리디하게 접근할 수 있나?
    - 일단 예제는 모두 되긴 하는데, 반례가 존재할 수 있다
   - 일단 해보자

### 💡 핵심 로직
- 그리디하게 피보나치 수를 선택하면 문제 조건을 만족할 수 있다.

### 📌 배운 점
- 그리디하게 문제를 풀면 최적해가 된다는 것을 증명할 수는 없으나, 시간복잡도를 바탕으로 문제를 해결하는 역량을 길렀습니다.
