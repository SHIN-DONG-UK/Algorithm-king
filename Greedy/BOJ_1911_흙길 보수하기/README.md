## 💡 BOJ 1911 - 흙길 보수하기 [[문제 링크](https://www.acmicpc.net/problem/1911)]

| 항목 | 내용 |
|------|------|
| 문제 유형 | `Greedy` |
| 사용 알고리즘 |  |
| 난이도 | `골드5` |

### ✅ 문제 설명

**[문제 요약]**

- N개의 물웅덩이가 있다
- 물웅덩이를 덮을 수 있는 길이가 L인 널빤지가 충분히 있다
- 물웅덩이들의 위치와 크기에 대한 정보가 주어질 때, 모든 물웅덩이를 덮기 위해 필요한 널빤지들의 최소 개수를 구하라

**[제약 사항]**

- 1 <= N <= 10,000
- 1 <= L <= 1,000,000
- 웅덩이들은 서로 겹치지 않는다
- 웅덩이의 시작 위치와 끝 위치는 0이상 10억 이하

### 🔍 풀이 접근

- 두 웅덩이를 같이 덮을지 말지를 결정하는게 중요하다
- k번 웅덩이와 k+1번 웅덩이를 같이 덮은 것이 뒤의 결과에 영향을 끼치는가?
- 같이 덮음으로 인해 널빤지의 시작 위치가 밀릴 수는 있다
- 그런데 시작 위치가 밀리는게 무조건 좋은것이므로, 크게 문제가 되지 않는다
- 따라서 앞에서부터 같이 덮는게 좋으면 같이 덮고, 그렇지 않으면 새로 덮는 그리디 알고리즘을 적용해보자

### 💡 핵심 로직

- 물웅덩이를 시작위치를 기준으로 오름차순 정렬한다
- 정렬된 물웅덩이를 순서대로 돈다
    - 널빤지의 마지막 위치가 현재 물웅덩이의 끝 위치보다 크다 -> 이미 덮인 것이므로 넘어간다
    - 시작 위치보다 크다 -> 같이 덮는게 무조건 좋다
    - 시작 위치보다 작다 -> 널빤지를 새로 시작한다

### 📌 배운 점
- 문제 상황을 이해하고 그리디하게 접근하는 것이 최적해임을 이해하고 문제를 풀이하는 경험을 했습니다.
